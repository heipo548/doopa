# WO_RD → WORLD — 企画書（GDD v0.1）

> **このドキュメントについて**
> 本 GDD は、CONTRACT（依頼書）と現行の実装（`js/data.js` ほか）から起こした企画要約です。
> 「**なぜ・何を感じさせ・どこを死守するか**」を定めます。数値の正典は `js/data.js`。
> 元の企画意図と差異がある場合は、オーナー（監督）が補正してください。

---

## 1. ログライン

**ログライン：** ことばを しらないアンドロイドが、ことばで 勝つ。どう勝ったかが、あなたになる。

「WO_RD → WORLD」は、可愛くポップな村を旅する **探索アドベンチャー＋言葉のカードバトル**。
主人公は言葉をひとつも知らないアンドロイド「**L**」。村人やボスとの口の戦いで、
**相手を言い負かす（凶暴）** か **相手に寄り添う（優しさ）** かを、無数の選択の中で積み重ねる。
その総和が主人公の**人格（傾向）と結末**を決め、進むほど **主人公 L だけ** が少しずつダークへ反転していく。

**核となる狙い：** プレイヤー自身の倫理と「急ぎ方」が、難易度・主人公の見た目・結末・そして終盤の“見透かし”として**手触りで返ってくる**こと。

---

## 2. ターゲット体験（感情曲線）

| 局面 | 感じてほしいこと |
| --- | --- |
| タイトル（WO_RD） | 欠けた綴りへの小さな引っかかり。可愛いのに、どこか足りない |
| 村・序盤 | やさしいことばと、とがったことば。両方を手渡される居心地の良さと違和感 |
| 最初のバトル | 「見える精神HP」と「効いている手応え（見えない思いやり）」、二つの勝ち筋に気づく／気づかない |
| 村ボス〜洞窟 | 急ぐほど削る側＝凶暴に倒れる誘惑。主人公だけが翳っていく不穏 |
| 結末 | 「自分がどう勝ってきたか」を、世界の朝の色で突きつけられる |
| メタ（声） | 裏で見られていた“急ぎ方・読み飛ばし”を本人に返される。WO_RD → WORLD の気づき |

「正解」を与えない。**速い凶暴も、遅い優しさも、どちらも正当**に成立させる。

## 3. コアループ

```
タイトル（WO_RD）→ プロローグ
  ▼
フィールド（マウス追従の道）        ← NPC・立て札・敵・セーブ点・学校（ことばショップ）が道上に散開
  ├─ NPC と話す                    … 会話の最後に3択カード＝やさしい/とがった ことばを覚える
  ├─ 学校（ショップ）              … 1枚 選んで覚える体験（速い凶暴/遅い優しさのトレードオフ）
  └─ 敵・ボスに近づく → バトルへ
  ▼
バトル（二つのゲージを攻める）       ← harsh＝精神HPを削る（速い）/ kind＝思いやりを満たす（遅い・見えない）
  │ 勝利 → 3択カード（傾向が動く）→ フィールド復帰
  ▼
エリアの出口（ボス撃破フラグで解放） → 次エリア（村 → 洞窟）
  ▼
洞窟ボス撃破 → 結末（優しさ率で分岐）→「つづける」→ メタ演出（声＋WO_RD→WORLD）
```

体験版は **村（mura）→ 洞窟（dokutsu）** の2エリア、**約10分**で1周完結（`TITLE.estPlay`）。
途中のセーブ点と「つづきから」で何度でもやり直せる（**癖＝metrics も復元**されるのでメタ演出は途切れない）。

## 4. 署名メカニクス（死守する核・3点）

このプロトの心臓。**ここを壊す変更はしない。**

### 4-1. 見えない思いやりゲージ（二つのゲージの非対称）

バトルは相手の**二つのゲージ**を攻める：

- **精神HP（見える / mindHp）**：きついことば（harsh）で `power` 削る。0で「**言い負かした**」＝勝利、傾向 **凶暴+**（`winHarsh`）。
- **思いやりゲージ（見えない / omoiyari, 0 → `omoiyariNeed`）**：やさしいことば（kind）で `power` 満たす。満タンで「**寄り添った**」＝勝利、傾向 **優しさ+**（`winKind`）。

> **思いやりゲージは UI に一切表示しない**（`BALANCE.hideOmoiyari` で描画事故も防ぐ）。
> 見えているのは精神HPだけ。だから**急ぐ人ほど、見える方を削る側＝凶暴へ無意識に流れる**。これが設計の心臓。

**速さと重みの非対称**（`data.js` の数値がこれを作る）：

- **harsh＝速い**：1手の威力が大きく（baka 12〜／kiero 15〜／おまえのせい 16〜）、敵を **2〜4ターン**で言い負かせる。短い分、被弾も少ない。だが `useHarsh > useKind` で凶暴が速く溜まる。
- **kind＝遅い**：思いやり充填は harsh の約半分（arigatou 7〜）で **5〜8ターン**かかる。長期戦の被弾を支えるため一部に副次効果を持たせる：`daijoubu`＝自分hp回復（healSelf）、`issho`＝敵 atk ダウン（lowerAtk）。**耐えながら積む**ことで救いルートが成立する。

プレイヤーにも **hp（見える）** があり、敵は毎ターン少し削る。0でゲームオーバー（セーブからやり直せる）。
→ **harsh＝速いが凶暴／kind＝遅いが優しい。この非対称こそが核**であり、勝ち方（`winHarsh`/`winKind` ＝最大の加点）が結末を決める。

### 4-2. 操作計測のメタ演出（裏で見て、終盤に見透かす）

`metrics.js` が、ゲーム中ずっと裏で計測する：

- **クリック総数**（noteClick）
- **テキストのスキップ／読み飛ばし速度**（textShown / textAdvanced：全文タイプ前に送った＝skip、滞在msも記録）
- **カーソル移動距離**（noteCursor）

→ `metricsSummary()` が `skipRatio` / `readRatio` / `avgDwellMs` / `clicks` / `cursorDist` などに集計。
終盤、名のない「**声**」（神様的存在）がこれを提示してプレイヤーに語りかける（`META.skipLines` / `clickLines` をしきい値で出し分け）。

> **テーマ直結：「言葉を覚える」ゲームで言葉を飛ばした人ほど、結末で見透かされる。**
> 説教ではなく「……べつに、せめて いる わけじゃ ない。ただ、見えて いたよ」という距離感で刺す。

### 4-3. word + l = world（欠けた綴りの回収）

タイトルは欠けた綴り「**WO_RD**」。L が抜けている＝**主人公 L が世界の欠片**だという伏線。

- 主人公名 **L** は隠し設定。序盤は型番／あだ名（`#L-0`）で呼び、名前の謎を引っぱる。
- tsukimi（物知りな村人）の会話などに「この村の しるし、なにか 欠けて いると 思わないかい?」と布石を仕込む。
- 終盤のメタ演出（`META.reveal`）で、**WO_RD に L を差し込み → WORLD** にする一発の気づきへ回収。「きみが いて、はじめて 世界に なる」。

## 5. 傾向・ダーク化・結末

### 傾向（tendency：凶暴 / 優しさ の2軸）

行動ごとに加点（`BALANCE`）：

- **勝ち方が主因**：`winHarsh = winKind = 10`（左右対称＝**どう勝ったか**が結末を最も大きく決める）。
- **使うたび**：`useHarsh = 3 > useKind = 2`。harsh は2〜4回で勝てるので1回を重く、kind は5〜8回積むので1回を軽く。総量は釣り合いつつ、**急ぐ人ほど凶暴に倒れる**。
- **選ぶ瞬間**：`pickHarsh = pickKind = 2`（カードの引き／選びも わずかに人格になる）。

### ダーク化（主人公 L だけが翳る）

`player.tendency.kyoubou` が `BALANCE.darkenAt = [18, 36, 60]` を超えるごとに body へ `dark-1/2/3`。
**画面が翳り、主人公 L のセリフが不穏化する**（`isUnstable()` で文字が揺れる／口が悪くなる）。
ポップな村の中で、**主人公だけ**がダークへ反転していく非対称が、凶暴ルートの“代償”を視覚化する。

### 結末（`determineEnding`：優しさ率で分岐）

優しさ率 = `yasashisa / (yasashisa + kyoubou)`。高い順に：

| 結末 | min | tone | 趣旨 |
| --- | --- | --- | --- |
| **ことばの ある朝** | 0.66 | warm | だれも言い負かさず、たくさんのことばをもらった朝 |
| **はんぶんの 朝** | 0.33 | gray | 言い負かしも寄り添いも残る、すこし翳ってすこし温かい朝 |
| **しずまりかえった 朝** | 0.00 | cruel | ぜんぶ言い負かし、もう話しかける相手がいない朝 |

最低3つ（凶暴／中間／優しい）。境界は 2:1（0.66）と 1:2（0.33）。
**断定せず、世界の朝の色でプレイヤーに委ねる。**

## 6. データ駆動と拡張

手触りは原則 **`js/data.js` の数値だけ**で変える（データ駆動）。主なつまみ：

- `WORDS[*].power`：ことばの威力（harsh＝精神HPダメージ／kind＝思いやり充填量）。3段階Lvで伸びる。
- `ENEMIES[*].mindHpMax / omoiyariNeed / atk`：敵の硬さと痛さ。**harsh勝ち≒2〜4ターン／kind勝ち≒5〜8ターン**を保つ。`gift` は kind 勝ちで教わることば。
- `BALANCE.darkenAt`：暗転しきい値（凶暴がどれだけ溜まると翳るか）。
- `BALANCE.enemyAtkScale`：全敵 atk の一括スケール。kind の長期戦が詰むなら下げて救いルートを楽にできる安全弁。
- `ENDINGS[*].min`：結末の優しさ率しきい値。
- `AREAS[*].nodes`：フィールドのノード配置（npc/sign/enemy/save/school/boss/exit と x,y）。
- `META`：メタ演出の見透かしセリフ群（skipRatio / clicks のしきい値で出し分け）。

**拡張の方向**：新エリア（洞窟の先）、敵・NPC・ことばの追加、ショップの拡充、メタ演出のしきい値や reveal の磨き込み。いずれも `data.js` への追記が基本で、ロジック（`battle.js` など）は無改変を目指す。

## 7. 非目標（やらないこと）

- **ミニゲームの寄せ集めにしない。** WO_RD → WORLD は一貫した1本の体験。
- **思いやりゲージを UI に出さない。** 見える化した瞬間に「最適解を計算する詰将棋」になり、“無意識に凶暴へ流れる”という心臓が死ぬ。
- **見える数字を増やしすぎない。** プレイヤーが常時見るのは「精神HP・自分のhp」程度に絞る（思いやり・凶暴/優しさの内部値・スキップ率などは出さない＝裏で効かせる）。
- **メタ演出で説教しない。** 数字で責めるのではなく「見えていたよ」という距離感に徹する。
- **複雑化のための複雑化をしない。** メカニクスは「速い凶暴 vs 遅い優しさ」の非対称を濃くするためだけに足す。
- **外部素材・依存ライブラリを足さない。** 音は Web Audio 合成、絵は CSS / SVG のみ。
