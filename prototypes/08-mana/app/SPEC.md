## 1. ゴールと完成定義（DoD）

**ゴール**: 「名付ける」という単一の動詞が、謎解き・ボス戦・物語・演出のすべてを駆動することを、
初見プレイヤーが **20〜25分** で体験できるデモ。

1. タイトル→名前入力→プロローグ→村（パズルP0〜P4）→ボス→エンドカードまで、操作不能バグなしで通しプレイできる
2. 初見プレイヤーがヒントシステム（きき診3段階）だけで詰まずにクリアできる
3. 命名成功時にモノクロ→彩色演出が機能し、ボス解決時に画面全体が彩色される
4. `npm run dev` で起動、`npm run build` で頒布可能な静的ビルドが生成される
5. RecipeEngine にユニットテストがあり `npm test` が通る

## 2. スコープ

**IN**: Boot / Preload / Title（名前入力）/ Prologue / World / NamingOverlay / Dialogue / EndCard、
パズル P0(火) P1(明) P2(木・休) P3(犬) P4(凪)、シークレット S1(鳥) S2(玉) S3(水)、
ボス戦（火のナナシ・3フェーズ）、きき診ヒント、言ノ葉インベントリ（消費なし）、
結字・改字・単字命名、彩色演出、セーブ（休ベンチ）、会話。

多言語対応、ゲームパッド、モバイルビルド、外部画像・音声アセットのダウンロード。

```
Boot → Preload(フォント/テクスチャ) → Title（名前入力 最大6文字 デフォルト「ヒビキ」/ つづきから）

```

DialogueScene は World 起動時に常駐 launch。NamingOverlayScene は名付け時のみ launch し、
表示中は World を `scene.pause`（時間が止まる演出を兼ねる）。

## 6. 操作仕様

| 入力 | 動作 |
|---|---|
| WASD / 矢印 | 移動（160px/s・8方向） |
| E / Z | 調べる・話す・拾う・鐘を鳴らす（近接 半径48px） |
| Q / X 長押し0.6s | きき診（3段階ヒント・使うたび段階が進む） |
| Space | 名付けモード（命名可能対象の近くのみ。対象頭上に「？」マーカー） |
| Esc | オーバーレイを閉じる |
| マウス/タップ | 名付けオーバーレイ内の選択・決定（UIは全てクリック可能） |

会話中・名付け中は移動入力を遮断。インタラクト優先度: 会話 > 拾う > 調べる。

## 7. コアシステム

### 7-1 状態機械
`UNNAMED（ナナシ・墨ノイズ）--正しい名付け--> NAMED（実体化・彩色）`。
無効な名付けはソフト失敗（「……なにもおこらない」）。失敗は即ゲームオーバーにしない。

### 7-2 名付けの3様式
1. **単字**: 対象に base 字がない。語彙から1字。例: 火→竈
2. **結字**: スロット2。2字を結び、成立すると **新しい字が語彙に永続追加**。例: 日＋月＝明
3. **改字**: 対象に base 字があり、きき診（段階2以上）で正体判明後に解禁。
   対象の字が中央に大きく坐り、修飾字（点・止・一）を1つ打ち込む。例: 大＋点＝犬

**言ノ葉は消費しない。** 語彙（知っている字）がそのまま能力＝成長。

### 7-3 RecipeEngine（純粋関数・テスト必須）
```
resolve(inputs: string[], targetBase?: string, targetTags?: string[]): {glyph, recipeId} | null
解決順序: ① targetBase あり→改字テーブル（不一致なら base+修飾字で結字テーブルへフォールバック）
　　　　　② 2字→結字テーブル（順不同一致・system 除外）
　　　　　③ 1字→単字テーブル（対象タグ一致）
　　　　　④ null（ソフト失敗）
```

### 7-4 EffectRegistry
`apply(glyph, targetId, world)` で副作用を発火。新規 effect は表への登録のみで完結。

### 7-5 HintSystem（きき診）
対象ごとに3段階（性質→部首→ほぼ答え）。段階は対象ごとにセーブ。
ボス戦のみ被弾3回でコトノ婆の叫びアシストを追加表示。

### 7-6 DialogueSystem
`{speaker, text}` のキュー。タイプライター30字/秒。E で全文表示→次へ。話者色 §13。

### 7-7 SaveSystem
休ベンチ使用時に保存: `{name, flags, knownGlyphs, gems, pos, playtime, hintLevels}`。
タイトルに「つづきから」（セーブ存在時のみ）。破損 JSON は新規データへフォールバック。

### 7-8 ColorSystem
全表示物は `{mono, color}` の2プレート。`colorize` で0.8秒クロスフェード＋墨はじけ8粒。
ボス解決時 `colorizeAll`（カメラが村をパン・約4秒）→白フラッシュ→環境音「無音→鳥と風」。

## 8. データ定義（正は src/data/。以下は要約）

### 8-2 レシピ

改字: 大+点=犬, 風+止=凪, 烏+点=鳥, 王+点=玉, 木+一=本
単字: 火(tag:hearth), 木(tag:driftwood), 水(tag:well)
システム: 火+火=炎（プレイヤー使用不可・ボスP2演出専用）

### 8-3 主要エンティティ
竈(P0) / くらやみ(P1) / 流木→橋(P2) / 休み処台座×2 / 大きな影→犬(P3) / 嵐の壁→凪(P4) /

社の鐘 / 村の名標（灯で自動解決→「灯し村」） / 壊れた水車 / 掘り跡×2（犬の近道）

## 9. レベル「灯し村」

ワールド 2880×1620。ゾーン: 火の広場(START) / 犬の道・丘 / 風の丘 / 社 / 東通路 /
暗がりの間 / 南の道 / 川辺・東岸。主要座標は `data/level_village.ts` が正。

進行: P0(火)→点入手 → P1(明)とP2(木)は順不同 → P3(犬・物理的に影が北の道をふさぐ)
→ P4(凪・止は風の丘) → 社（ボス）。
順序はフラグゲートでなく「修飾字の入手順」で自然に守られる。

| ID | 状況 | 正解 | 報酬 |
|---|---|---|---|
| P0 | 竈のナナシ・寒い | 火を拾い単字「火」 | 点・コトノ婆会話 |
| P1 | 暗がりの間 | 日+月→結字「明」 | 追従光球・闇マスク解除 |
| P2 | 川を渡れない / 休めない | 流木に「木」/ 台座に人+木「休」 | 渡河・セーブ解禁 |
| P3 | 巨大な影が道をふさぐ | きき診→大+点「犬」 | 犬が仲間に・掘り跡近道 |
| P4 | 嵐で社へ行けない | 風+止「凪」 | 社への道 |
| S1 | 烏が泣いている | 烏+点「鳥」 | 玉+1・井戸の在処 |
| S2 | 王の像 | 王+点「玉」 | 玉+1 |
| S3 | 涸れ井戸 | 水を入手し単字「水」 | 玉+1・村人B彩色 |

闇の奥（x>1800）に明なしで進むと入口へ戻される。嵐ゾーンは凪まで風圧で押し戻される。

## 10. ボス戦 — 火のナナシ（社）

```
INTRO: 接近で発火演出＋会話3行 → FIGHT_P1
P1: 体当たり突進（速度200・1.5s間隔）。被弾はノックバック＋赤フラッシュ（ダメージ数値なし）。
    きき診可・名付け可（正解「火」単字）→「名は当たり…されど居場所がない！」→ P2
P2: 炎（火+火）に分裂・放射状の火の粒弾（8方向・速度120・2s毎）。被弾3回でコトノ婆アシスト。
    社の鐘に E →ゴーン→5秒の鎮み（初回のみ「丁」落下）。
    鎮み中に名付け→改字UI（火＋丁）or 結字（火+丁）→「灯」→ RESOLVE
RESOLVE: 時止め→「灯」結晶化→炎が灯籠に宿る→ colorizeAll →名標「灯し村」→
    妹の名スロット「灯◇」点灯→「……にいちゃん？」→ EndCard
```
戦闘中はアリーナ西の門が閉じる。ベンチ休憩は不可。

## 14. アート/オーディオ プレースホルダ方針（権利クリーン縛り）

**画像ファイルを一切使わない。** 全アセットは (a) Phaser Graphics、(b) Shippori Mincho で
描画した文字→テクスチャ、(c) パーティクルで構成。音はすべて Web Audio 生成。
プレイヤー＝白い円＋笠。犬＝「犬」の字が小さく実体化した姿。NPC＝灰シルエット→彩色。
使用 AI ツール・ライブラリは README「開発ツール開示」に記録（コンテスト AI 利用ガイドライン対応）。
