# さいごのともだち — ことばで、たたかう夜（口喧嘩バトル PoC v0.3）

「さいごのともだち」の **コア戦闘プロトタイプ** です。武器ではなく **ことば** で戦う口喧嘩バトル。
ひと晩（全6ウェーブ＋ボス）を生き延びる中で、群れに **ことばを ぶつける** か **手をのばして むかえる** かを毎ターン選びます。
その選択の比率（むかえた率＝救済率）で、夜明けの結末が変わります。

> ビルド不要・依存パッケージなし。`index.html` をブラウザで開くだけで動きます。
>
> **核：手が止まらないのに、心が止まる。そして、言えなかった言葉が、言えるようになる。**

---

## 遊び方（クリックだけで進みます）

画面下のコマンドバーから選びます。基本は **コマンド → （ことば/対象）を選ぶ** の2ステップです。

| コマンド | 何をする | 性質 |
| --- | --- | --- |
| **ぶつける** | きついことば（とげとげ）を相手にぶつける | 早く退けられる（EXP多）。でも **狂気**が上がり、二度と会えない |
| **きいてみる** | 相手の声をきく。「心の壁」を下げる | 敵ごとに効くことばが違う（手順パズル）。こころを消費 |
| **手をのばす** | 壁が0の相手を こちら側に むかえる | むかえた+1・HP回復・**その子のことばを教わる**。遅いが誰も失わない |
| **もちもの** | おまもりでHP回復 | 所持数に限りあり |
| **こらえる** | ことばを飲み込んで耐える | このターンの被ダメを大幅カット |

> 言ったことばは画面に**吹き出し**で出ます（とげとげ＝不穏／やさしい＝灯り／問いかけ＝水色／沈黙＝かすれ）。

### 勝ち筋は2つ（非対称なのが核）

- **ぶつけるルート（速い・易しい）**：群れを素早く退けるほど被弾が減る。狂気でことばが鋭くなり早く強くなれるが、むかえた率0%＝結末は **しずかな朝**（最強だが空っぽ）。
- **手をのばすルート（遅い・難しい）**：壁を下げると相手は *おだやかになり攻撃をやめる*。むかえるとHPも回復＆**ぬくもり**で最大HPが伸びる。慈悲を重ねるほど夜を生き延びられる。高救済率＝結末は **ひかりの朝**。

### 語彙が増える（言えることが増える成長）

- 主人公は最初、言えることばが少ない。戦い・救済・3択を通じて **やさしいことばを覚えていく**。
- 友を **手をのばして** むかえると、その子の口ぐせを教わる（例：くろまる→「ありがとう」）。
- 3択の「おぼえる：◯◯」カードでも増える。覚えたことばは **きいてみる でいつでも使える**。
- 覚えたことばは画面下の「**ことば図鑑（やさしい）**」に並びます。

### こころ / 心の壁 / おだやか

- **こころ**：きく/むかえる の燃料。`きいてみる`（1）と `手をのばす`（2）で消費。**毎ウェーブ満タンに回復**。
- **心の壁（♥）**：その相手を むかえるのに必要な「きいてみる」回数。**効くことばでだけ** 下がる。
- 壁が **0（すくえそう）** になると、その相手は **もう攻撃してこない**。だから「壁を下げる」こと自体が生存手段になります。

### きもち（とげとげ ↔ やさしい の1本バー）

ことばの積み方が、中央から左右に傾く1本の「きもち」バーに出ます（要素過多をへらすため2本→1本に統合）。

- **とげとげ（左）**：きついことばで傾く。傾くほど ことばダメージ↑・画面が翳る。
- **やさしい（右）**：やさしいことばで傾く。たまるごとに **最大HP＋＆回復**＝救いルートの雪だるま。

### レベルアップ（3択）

ウェーブをクリアするたびに3択カードが出ます。新しいことば・ことば強化・最大HP/こころ・パッシブを選んでビルドを伸ばします。
**ことばのLvが最大＋対応パッシブ** で ことばが進化します（下表）。

| ことば | 進化キー（パッシブ） | 進化後 |
| --- | --- | --- |
| バカ（単体） | ふかいかなしみ | もう みんな きらい（全体5＋壁−1） |
| うるさい（前列2） | おこりんぼ | しつこいしつこい！（前列3に6） |
| あっちいけ（全体） | ねがい | いっしょに かえろう（全体の壁−2＋**ぬくもり＋**） |

> **いっしょに かえろう**（旧・救済の光）は、きついことばが **反転して** 群れ全員へ手をのばす進化。狂気を積まず、救いルートの要です。

## 起動方法

### A. いちばん簡単（推奨）
`index.html` をダブルクリックしてブラウザで開くだけ。
（ES Modules を使わず素の `<script>` で書いているので、`file://` でも動きます。）

### B. ローカルサーバで開く
一部ブラウザの制約を避けたい場合：

```sh
cd ~/Projects/ai-workspace/99_sandbox/doopa/prototypes/02-saigo-no-tomodachi
python3 -m http.server 8765
# → ブラウザで http://localhost:8765 を開く
```

音（効果音・BGM）は Web Audio API による合成音です。外部音源ファイルは不要。
タイトルの「はじめる」を押した後に鳴ります（ブラウザの自動再生制約のため）。右上の ♪ かキー **M** でON/OFF。

## 数値の調整方法（バランス調整）

ゲームの手触りは **`js/data.js` の数値だけ** で変えられます（データ駆動）。代表的なつまみ：

- `PLAYER_INIT.maxHp / maxKokoro / items.heal`：開始時の体力・こころ・回復アイテム数。
- `BALANCE`：
  - `healOnWaveStartPct`：ウェーブ開始時のHP回復割合（`0`で回復なし＝高難度、`1.0`で毎波全快）。
  - `saveHeal`：すくう成功で回復するHP（救いルートの持続力）。
  - `actCost / saveCost`：こころみる・すくうのこころ消費。
  - `expToBonusLevel`：祓ってEXPが貯まると“もう1回3択”が出る量（祓い＝早く強くの表現）。
  - `eliteHpBonus / eliteAtkBonus`：ウェーブ5強化個体の上乗せ。
- `WEAPONS[*].power / perLv`：武器威力（`実威力 = power + (Lv-1)*perLv`、進化武器はLvなし）。
- `ENEMIES[*].hp / atk / wall / acts`：敵の体力・攻撃・心の壁・効くACT。
- `WAVES`：各ウェーブの敵構成。`ENDINGS[*].min`：結末の救済率しきい値。

## 動作確認（ヘッドレス検証）

このPoCは Node を使わず、macOS標準の JavaScriptエンジン（`osascript -l JavaScript`）で検証します。
カレントは prototype ルート（このフォルダ）を想定：

```sh
# コア（data/state/battle/cards）の正当性＋語彙・ゲージのユニットテスト
osascript -l JavaScript tools/headless-check.js
# 簡易AIで“ぶつける寄り／手をのばす寄り”を各20ラン回し、両ルートの成立を確認
osascript -l JavaScript tools/balance-probe.js
# DOM依存（ui/audio/main）の参照チェック（document/window スタブ）
osascript -l JavaScript tools/ui-syntax-check.js
```

- **正当性テスト**：FSMが最後まで回る／進化3種＋未達では進化しない／結末しきい値／ボス撃破・救済→夜明け／効かないことばは壁を下げない … 全PASS。
- **語彙・ゲージ**：狂気でことばダメージ↑／きついことばで狂気↑／ぬくもり閾値で最大HP↑／むかえると gift のことばを習得／やさしい汎用語は壁を割らずぬくもり＋ … 全PASS。
- **バランス探針**（各20ラン×簡易AI）：ぶつける寄りは W6到達 約7.5割・結末 **しずかな朝**／手をのばす寄りは 約9割・結末 **ひかりの朝**。非対称が保たれ両ルート成立。

> ブラウザ実機での**目視（クリック操作・見た目・音）は未確認**。`index.html` を開く（または下のローカルサーバ）と最終確認できます。特に ドット絵シーン・吹き出し・2ゲージの見栄えは要・人の目チェック。

## 既知の調整余地（次に触るなら）

- 狂気のダメージ倍率（`kyokiDmgScale`）・ぬくもり閾値（`nukumoriStep`）は**暫定**。実機の手触りで詰める。
- 序盤（ウェーブ3〜4）の難度はやや高め。`healOnWaveStartPct` や `maxHp` で微調整可能。
- 手をのばすルートは「いっしょに かえろう」入手前がいちばん苦しい。`CARDS` の重みづけ（出現確率）導入も一案。
- 内部の関数名・データ id は v0.1 のまま（`cmdFight`/`WEAPONS`/`namida` など）。段階的なリネームは将来課題。
- 数値はすべて暫定。実機で遊んだ感触に合わせて `data.js` を詰めるのがおすすめです。
