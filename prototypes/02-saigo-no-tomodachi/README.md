# さいごのともだち — Core Combat PoC

コマンド選択型サバイバーRPG「さいごのともだち」の **コア戦闘プロトタイプ（v0.1）** です。
ひと晩（全6ウェーブ＋ボス）を生き延びる中で、群れを **祓う（倒す）** か **救う** かを毎ターン選びます。
その選択の比率（救済率）で、夜明けの結末が変わります。

> ビルド不要・依存パッケージなし。`index.html` をブラウザで開くだけで動きます。

---

## 遊び方（クリックだけで進みます）

画面下のコマンドバーから選びます。基本は **コマンド → （武器/ACT/対象）を選ぶ** の2ステップです。

| コマンド | 何をする | 性質 |
| --- | --- | --- |
| **祓う** | 武器で攻撃して倒す | 1ターンで即・強くなる（EXP多）。でも二度と会えない |
| **こころみる** | 「心の壁」を下げる | 敵ごとに効くやり方が違う（手順パズル）。こころを消費 |
| **すくう** | 壁が0の相手を救う | 救済+1・HPも少し回復。遅くて弱いが、誰も死なない |
| **どうぐ** | なみだのおまもりでHP回復 | 所持数に限りあり |
| **まもる** | このターンの被ダメを大幅カット | 全体攻撃のしのぎ手段 |

### 勝ち筋は2つ（非対称なのが核）

- **祓いルート（速い・易しい）**：群れを素早く消すほど被弾が減る。早く強くなれるが、救済率0%＝結末は **しずかな朝**（最強だが空っぽ）。
- **救いルート（遅い・難しい）**：壁を下げると敵は *おだやかになり攻撃をやめる*。救うとHPも回復するので、慈悲を重ねるほど夜を生き延びられる。高救済率＝結末は **ひかりの朝**。

### こころ / 心の壁 / おだやか

- **こころ**：救済リソース。`こころみる`（1）と `すくう`（2）で消費。**毎ウェーブ満タンに回復**。
- **心の壁（♥）**：その敵を救うのに必要な「こころみる」回数。**効くACTでだけ** 下がる。
- 壁が **0（すくえそう）** になると、その敵は **もう攻撃してこない**。だから「壁を下げる」こと自体が生存手段になります。

### レベルアップ（3択）

ウェーブをクリアするたびに3択カードが出ます。新武器・武器強化・最大HP/こころ・パッシブを選んでビルドを伸ばします。
**武器Lvが最大＋対応パッシブ** で武器が進化します（下表）。

| 武器 | 進化キー（パッシブ） | 進化後 |
| --- | --- | --- |
| なみだ砲（単体） | ふかいかなしみ | 大こうずい（全体6＋壁−1） |
| ぽよぽよ連打（前列2） | おこりんぼ | ばくれつぽよ（前列3に6） |
| ひかりの輪（全体） | ねがい | 救済の光（全体2＋**壁−2**） |

> **救済の光** は全体の壁を一気に下げる “一斉なだめ” エンジン。救いルートの要です。

## 起動方法

### A. いちばん簡単（推奨）
`index.html` をダブルクリックしてブラウザで開くだけ。
（ES Modules を使わず素の `<script>` で書いているので、`file://` でも動きます。）

### B. ローカルサーバで開く
一部ブラウザの制約を避けたい場合：

```sh
cd ~/Projects/ai-workspace/99_sandbox/saigo-no-tomodachi
python3 -m http.server 8765
# → ブラウザで http://localhost:8765 を開く
```

音（効果音・BGM）は Web Audio API による合成音です。外部音源ファイルは不要。
タイトルの「はじめる」を押した後に鳴ります（ブラウザの自動再生制約のため）。右上の ♪ かキー **M** でON/OFF。

## 数値の調整方法（バランス調整）

ゲームの手触りは **`js/data.js` の数値だけ** で変えられます（データ駆動）。代表的なつまみ：

- `PLAYER_INIT.maxHp / maxKokoro / items.heal`：開始時の体力・こころ・回復アイテム数。
- `BALANCE`：
  - `healOnWaveStartPct`：ウェーブ開始時のHP回復割合（`0`で回復なし＝高難度、`1.0`で毎波全快）。
  - `saveHeal`：すくう成功で回復するHP（救いルートの持続力）。
  - `actCost / saveCost`：こころみる・すくうのこころ消費。
  - `expToBonusLevel`：祓ってEXPが貯まると“もう1回3択”が出る量（祓い＝早く強くの表現）。
  - `eliteHpBonus / eliteAtkBonus`：ウェーブ5強化個体の上乗せ。
- `WEAPONS[*].power / perLv`：武器威力（`実威力 = power + (Lv-1)*perLv`、進化武器はLvなし）。
- `ENEMIES[*].hp / atk / wall / acts`：敵の体力・攻撃・心の壁・効くACT。
- `WAVES`：各ウェーブの敵構成。`ENDINGS[*].min`：結末の救済率しきい値。

## 動作確認（ヘッドレス検証）

このPoCは Node を使わず、macOS標準の JavaScriptエンジン（`osascript -l JavaScript`）で
DOM非依存のコア（data/state/battle/cards）を実際に走らせて検証しました。

- **正当性テスト**：FSMが最後まで回る／武器進化3種＋未達では進化しない／結末しきい値／
  ボス撃破→夜明け／ボス救済→夜明け／効かないACTは壁を下げない … 全PASS。
- **バランス探針**（各16ラン×簡易AI）で到達ウェーブ・救済率・結末を測定し、`data.js` を調整：
  - 祓いルート：ボス到達 約7割・結末 **しずかな朝**（速くて易しい）。
  - 救済特化（救済の光を完成）：ボス到達 16/16・救済率約99%・survive時は必ず **ひかりの朝**（難しいが成立）。
  - 中間の混成は **にび色の朝**。 → 三者三様の結末がプレイで分かれることを確認。

> ブラウザ実機での目視（クリック操作・画面確認）は未実施です。`index.html` を開いて手で遊ぶと、見た目・音・操作感を最終確認できます。

## 既知の調整余地（次に触るなら）

- 序盤（ウェーブ3〜4）の難度はやや高め。`healOnWaveStartPct` や `maxHp` で微調整可能。
- 救いルートは「救済の光」入手前がいちばん苦しい。3択の引きに左右されるので、`CARDS` の重みづけ（出現確率）導入も一案。
- 数値はすべて暫定。実機で遊んだ感触に合わせて `data.js` を詰めるのがおすすめです。
