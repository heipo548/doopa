# さいごのともだち — ことばで、たたかう夜（口喧嘩バトル＋夜の旅 PoC v0.4）

「さいごのともだち」の **プロトタイプ** です。武器ではなく **ことば** で戦う口喧嘩バトルを核に、
**街（拠点）⇄ 夜のフィールド（横スクロールの道）⇄ 戦闘 ⇄ 夜明けで街にかえる** という「いくつもの夜」の旅へひろげました（v0.4 メタループ first pass）。
夜の道を歩き、落ちていることばを拾い、救いを待つ友を むかえ、道のおわりで群れと むきあう——
むかえた友は街に住み、街は灯りを増やし、ぽちは「言えなかったこと」を すこしずつ 言えるようになります。

> ビルド不要・依存パッケージなし。`index.html` をブラウザで開くだけで動きます。
>
> **核：手が止まらないのに、心が止まる。そして、言えなかった言葉が、言えるようになる。**
>
> 物語の大枠は [`docs/STORY.md`](./docs/STORY.md)、企画は [`docs/GDD.md`](./docs/GDD.md)。

## 夜の旅（街 ⇄ フィールド ⇄ 戦闘 ⇄ 夜明け）— v0.4

「はじめる」を押すと、まず **街（拠点）** に出ます。そこから ひと晩を まわります。

```
（プロローグ）        ← はじめる の直後に物語の入口（スキップ可・マウスのみ）
  ▼
街（拠点）            ← 夜カウンタ・灯り（救った累計）・なかま・覚えたことば
  │ 「夜へ でかける」
  ▼
夜のフィールド（見下ろしの道）  ← 🖱 マウスカーソルに ぽちが自由追従（キーボード不要・縦横に歩ける）
  ├─ ☀ よふけメーター  … 滞在・救うほど「夜が濃く」なる。急ぐ＝浅い夜／長居・救い＝濃い夜
  ├─ 💬 おちている ことば … 通りかかると 拾う＝語彙＋1（その夜から使える・以後も持ち越し）
  ├─ 🙂 救いを待つ友     … 近づくと立ち止まり「こえをかける（むかえる）」/「とおりすぎる」。縦に避ければ素通りも
  └─ ⚔ 道のおわり        … 群れの けはい。「群れと むきあう」で戦闘へ
  │  （※夜が浅いほど＝急いだら 戦闘開始が有利／濃いほど＝救いを重ねたら 入りは苦しい＝先を急ぐ意味）
  ▼
戦闘（口喧嘩バトル：全6ウェーブ＋ボス）  ← これまで通り。下の「遊び方」参照
  │ 夜明け（救済率で その夜の結末）
  ▼
「🏠 街へ かえる」  → むかえた友が街に住み、灯りが増え、くらしのことば（おかえり/ただいま…）を授かる
  │ 夜カウンタ＋1
  ▼
… これを maxNights 回 …  → さいごの夜を越えると「ほんとうの夜明け」（マクロ結末）
```

- **持ち越し**：覚えたことばは **夜をまたいで消えません**（言えることは増え続ける）。街に住む友は、次の夜の **きずな**（群れの反撃を少し軽減）を そっと後押しします。
- **非対称はそのまま**：倒すだけの夜は 街が さみしいまま（くらしのことばは増えにくい）。救った夜だけ、帰り道が あたたかくなります。
- **よふけメーター（先を急ぐ意味）**：道に長居する・友を救うほど「夜が濃く」なり、その濃さが **戦闘の入り** に効きます（浅い夜＝こころ＋ぬくもりの好スタート／濃い夜＝こころ減＋世界が翳る）。でも救った友は街の灯り・きずな・語彙で **あとから必ず報われる**＝短期コスト／長期リターンの天秤。戦闘ロジック（`battle.js`）は無改変で、`field` が開始時のゲージにだけ載せます。
- 物語は **プロローグ＋初回ビート**（初めて道へ出る／友に会う／救う／灯りが増える…の一度きりの一言）で、遊びながら自然に伝わるようにしています。
- 夜の数・道の長さ・よふけの効きは `js/data.js` の `META` / `NIGHTFALL` で調整できます。

## 遊び方（クリックだけで進みます）

画面下のコマンドバーから選びます。基本は **コマンド → （ことば/対象）を選ぶ** の2ステップです。

| コマンド | 何をする | 性質 |
| --- | --- | --- |
| **ぶつける** | きついことば（とげとげ）を相手にぶつける | 早く退けられる（EXP多）。でも **狂気**が上がり、二度と会えない。**敵を直接クリックでも1タップ攻撃**（サクサク） |
| **きいてみる** | **群れ全体に話しかける**。効く相手の「心の壁」を下げる | みんなに一度に。敵ごとに効くことばが違う（💭が手がかり）。効いた子が **ことばを返す**（対話感）。こころを消費 |
| **手をのばす** | 壁が0の相手を こちら側に むかえる | むかえた+1・HP回復・**その子のことばを教わる**。遅いが誰も失わない |
| **もちもの** | おまもりでHP回復 | 所持数に限りあり |
| **こらえる** | ことばを飲み込んで耐える | このターンの被ダメを大幅カット |

> 言ったことばは画面に**吹き出し**で出ます（とげとげ＝不穏／やさしい＝灯り／問いかけ＝水色／沈黙＝かすれ）。

### 勝ち筋は2つ（非対称なのが核）

- **ぶつけるルート（速い・易しい）**：群れを素早く退けるほど被弾が減る。狂気でことばが鋭くなり早く強くなれるが、むかえた率0%＝結末は **しずかな朝**（最強だが空っぽ）。
- **手をのばすルート（遅い・難しい）**：壁を下げると相手は *おだやかになり攻撃をやめる*。むかえるとHPも回復＆**ぬくもり**で最大HPが伸びる。慈悲を重ねるほど夜を生き延びられる。高救済率＝結末は **ひかりの朝**。

### ネットミームの ことば（遊び心）

戦い・夜の道・3択を通じて、今っぽい **ミームのことば** も入手できます（知ってる人ほど ニヤッとする小演出つき）。
例：きついことば「**ま？**」「**草ァ！**」「**ざぁこ♡**」／やさしいことば「**わかる**」「**つよつよだね**」「**ぴえん……**」など。可愛い毒・共感の相づちとして、口喧嘩に温度を足します。

### 語彙が増える（言えることが増える成長）

- 主人公は最初、言えることばが少ない。戦い・救済・3択を通じて **やさしいことばを覚えていく**。
- 友を **手をのばして** むかえると、その子の口ぐせを教わる（例：くろまる→「ありがとう」）。
- 3択の「おぼえる：◯◯」カードでも増える。覚えたことばは **きいてみる でいつでも使える**。
- 覚えたことばは画面下の「**ことば図鑑（やさしい）**」に並びます。

### こころ / 心の壁 / おだやか

- **こころ**：きく/むかえる の燃料。`きいてみる`（1）と `手をのばす`（2）で消費。**毎ウェーブ満タンに回復**。
- **心の壁（♥）**：その相手を むかえるのに必要な「きいてみる」回数。**効くことばでだけ** 下がる。
- 壁が **0（すくえそう）** になると、その相手は **もう攻撃してこない**。だから「壁を下げる」こと自体が生存手段になります。

### きもち（とげとげ ↔ やさしい の1本バー）

ことばの積み方が、中央から左右に傾く1本の「きもち」バーに出ます（要素過多をへらすため2本→1本に統合）。

- **とげとげ（左）**：きついことばで傾く。傾くほど ことばダメージ↑・画面が翳る。
- **やさしい（右）**：やさしいことばで傾く。たまるごとに **最大HP＋＆回復**＝救いルートの雪だるま。

### レベルアップ（3択）

ウェーブをクリアするたびに3択カードが出ます。新しいことば・ことば強化・最大HP/こころ・パッシブを選んでビルドを伸ばします。
**ことばのLvが最大＋対応パッシブ** で ことばが進化します（下表）。

| ことば | 進化キー（パッシブ） | 進化後 |
| --- | --- | --- |
| バカ（単体） | ふかいかなしみ | もう みんな きらい（全体5＋壁−1） |
| うるさい（前列2） | おこりんぼ | しつこいしつこい！（前列3に6） |
| あっちいけ（全体） | ねがい | いっしょに かえろう（全体の壁−2＋**ぬくもり＋**） |

> **いっしょに かえろう**（旧・救済の光）は、きついことばが **反転して** 群れ全員へ手をのばす進化。狂気を積まず、救いルートの要です。

## 起動方法

### A. いちばん簡単（推奨）
`index.html` をダブルクリックしてブラウザで開くだけ。
（ES Modules を使わず素の `<script>` で書いているので、`file://` でも動きます。）

### B. ローカルサーバで開く
一部ブラウザの制約を避けたい場合：

```sh
cd ~/Projects/ai-workspace/99_sandbox/doopa/prototypes/02-saigo-no-tomodachi
python3 -m http.server 8765
# → ブラウザで http://localhost:8765 を開く
```

音（効果音・BGM）は Web Audio API による合成音です。外部音源ファイルは不要。
タイトルの「はじめる」を押した後に鳴ります（ブラウザの自動再生制約のため）。音のON/OFFは画面の **♪ ボタン**（戦闘は右上／街は右上）。

> 操作は **マウスのみ**（クリックと、夜の道での カーソル追従）。キーボードは使いません。

## 数値の調整方法（バランス調整）

ゲームの手触りは **`js/data.js` の数値だけ** で変えられます（データ駆動）。代表的なつまみ：

- `PLAYER_INIT.maxHp / maxKokoro / items.heal`：開始時の体力・こころ・回復アイテム数。
- `BALANCE`：
  - `healOnWaveStartPct`：ウェーブ開始時のHP回復割合（`0`で回復なし＝高難度、`1.0`で毎波全快）。
  - `saveHeal`：すくう成功で回復するHP（救いルートの持続力）。
  - `actCost / saveCost`：こころみる・すくうのこころ消費。
  - `expToBonusLevel`：祓ってEXPが貯まると“もう1回3択”が出る量（祓い＝早く強くの表現）。
  - `eliteHpBonus / eliteAtkBonus`：ウェーブ5強化個体の上乗せ。
- `WEAPONS[*].power / perLv`：武器威力（`実威力 = power + (Lv-1)*perLv`、進化武器はLvなし）。
- `ENEMIES[*].hp / atk / wall / acts`：敵の体力・攻撃・心の壁・効くACT。
- `WAVES`：各ウェーブの敵構成。`ENDINGS[*].min`：結末の救済率しきい値。

**v0.4（今回・メタループ first pass）**：
- **街（拠点）ハブ**：夜カウンタ・**灯り（救った友の累計＝街の明るさ）**・なかまの列・いえることば一覧・「夜へ でかける」。
- **夜のフィールド（見下ろしの道）**：🖱 マウス追従で **縦横に自由移動**（キーボード不使用）。**おちてることばを拾う**・**救いを待つ友を むかえる/とおりすぎる**・道のおわりで戦闘へ。
- **周回**：街→道→既存戦闘→夜明け→**街へかえる**（友が住む・灯り＋・くらしのことば授与）→…→さいごの夜→**マクロ結末**（全夜の救済総和で しずか/にび/ひかり）。
- **語彙の接続**：覚えたことばは**夜をまたいで持ち越し**。救った友が街にいるほど**くらしのことば**（おかえり/ただいま/おやすみ）を授かり、**きずな**を後押し（合計は上限でクランプ＝祓いを腐らせない）。
- **既存戦闘は無改変**（`battle.js` はそのまま）。街/道/結末の配線は `field.js`＋`ui.js`＋結果画面ボタンで実現。BGMに**街（やわらかい）/フィールド（前進感）**を追加。

**v0.6**：口喧嘩を“サクサクの対話”に（敵を直接クリック＝1タップ攻撃／きいてみる＝群れ全体に話す／効いた子が ことばを返す吹き出し）＋ネットミームのことば（ま？・草ァ！・ざぁこ♡／わかる・つよつよ…）と専用の小演出。

**v0.7（今回・「一択」の解消と直感UI＋フィールド自由化）**：
- **A. ことばに“重み”（こころコスト）**：*こころ* を「ことばを発する力（心のゆとり）」に一本化し、**ぶつけるにも こころを消費**（だまれ1／バカ2／きえろ・ま？3／全体3…）。**毎ターン自然回復**（こらえると多め）でハードロックしない。効率(バカ6/♡2)と一撃(きえろ7/♡3)の使い分け＝「最大威力の一択」を解消。**残酷さを吐くほど やさしくする余裕が減る**＝テーマ直結。
- **B. 対象1体は選択省略**：単体ことば／手をのばす とも、対象が1体なら 選ぶ手順を飛ばして即実行。
- **C. 文字を減らしアイコン駆動**：カード名から「おぼえる：」を全廃しアイコン＋ことばに。`⚡威力・♡こころ` のグリフで圧縮（直感UIの定石）。
- **D. フィールド自由化**：道のイベントを **2Dに散開**（友＝中央／ことば＝端）。道幅を広げ、端のことばは**縦に寄り道して拾う**＝寄るほど *よふけ* が濃くなる**リスク/リターン**。「右に行くだけ」を解消。
- **非対称ガード**：balance-probe で再測定し **harsh→しずか0%／gentle→ひかり100%** を3回とも維持・両ルート完走を確認。**見る数字は HP/こころ/きもち の3つに据え置き**（新ゲージは足さない＝分かりにくさ回避）。

**v0.8（今回・開発依頼書「最初の夜を短く・静かに・強く」P0＋P1）**：初見が“何を感じるべきか”を掴む前にシステム理解へ意識が向かないよう、**初回1分で核を体験**できるよう磨き込み。
- **P0-1 タイトルでHUD非表示**：`#game` を初期 `hidden` に。タイトルは題字＋寂しいコピー＋「はじめる」だけ（WAVE/数値/♪を出さない）。
- **P0-2 第一夜＝チュートリアル**：第一夜だけ `WAVES_TUTORIAL`（くろまる1体）に差し替え。最初は **ぶつける／きいてみる のみ**、**きいてみる で本音**（「……おいてかないで」）→ **手をのばす が解禁・脈動**。クリアで3択を出さず すぐ夜明け→街。道も短く静か（寄り道・よふけ無し）。
- **P0-3 用語整理**：操作語を **ぶつける／きいてみる／手をのばす** に集約。管理数値カウンタ（おいはらった/むかえた/きずな）を戦闘上部から撤去。**心の壁→こころのかべ／きずな→あかり／狂気→とげとげ／救済率は非表示**。よふけバーに補足。
- **P0-4 結果は街の景色で**：第一夜の帰還で **A=あかりが増える＋友＋「おかえり」習得／B=しずかなまま＋ぽち独白／C=きいた声が耳に残る** を、責めず・数値でなく 街で返す。
- **P1-1** 街の友はさわると生活セリフ（くろまる「ことば、まだ へた。でも、いる。」等）。**P1-2** とおりすぎた友に再会で記憶セリフ。**P1-3** 同じことばの使いすぎに小さな反応。**P1-4** こころ不足・解禁などの短い判断補助。
- 非対称（harsh→しずか0%／gentle→ひかり100%）は balance-probe で維持を再確認。JXA（headless/ui-syntax）緑。
- レビュー用エージェントを `.claude/agents/` に追加：**Toby Fox / 堀井雄二 / 桜井政博**（公開情報＋制作思想に着想・批評専用）。

**v0.9（実機FBで第一夜を“根本”作り直し／堀井雄二・桜井政博エージェント設計）**：初見FB（チュートリアル不明・戦闘がすぐ終わる・こころが分からない・街が欲しい・スマホで難）を体験ファーストで解決。
- **第一夜のくろまるは「ぶつけても消えない」**：一撃で倒して終わる出口を塞ぎ、「……あれ。きえ ない」→脈動が【きいてみる】へ移る→本音→【手をのばす】解禁、を**必ず指で通す**。コンセプト「きついことばしか知らないぽちが、きくを覚える」を“予想の裏切り”で体験させる（倒せないのは第一夜だけ・2夜目以降は通常）。

## 動作確認（ヘッドレス検証）

このPoCは Node を使わず、macOS標準の JavaScriptエンジン（`osascript -l JavaScript`）で検証します。
カレントは prototype ルート（このフォルダ）を想定：

```sh
# コア（data/state/battle/cards）の正当性＋語彙・ゲージのユニットテスト
osascript -l JavaScript tools/headless-check.js
# 簡易AIで“ぶつける寄り／手をのばす寄り”を各20ラン回し、両ルートの成立を確認
osascript -l JavaScript tools/balance-probe.js
# DOM依存（ui/audio/main）の参照チェック（document/window スタブ）
osascript -l JavaScript tools/ui-syntax-check.js
```

- **正当性テスト**：FSMが最後まで回る／進化3種＋未達では進化しない／結末しきい値／ボス撃破・救済→夜明け／効かないことばは壁を下げない … 全PASS。
- **語彙・ゲージ**：狂気でことばダメージ↑／きついことばで狂気↑／ぬくもり閾値で最大HP↑／むかえると gift のことばを習得／やさしい汎用語は壁を割らずぬくもり＋ … 全PASS。
- **バランス探針**（各20ラン×簡易AI）：ぶつける寄りは W6到達 約7.5割・結末 **しずかな朝**／手をのばす寄りは 約9割・結末 **ひかりの朝**。非対称が保たれ両ルート成立。

> ブラウザ実機での**目視（クリック操作・見た目・音）は未確認**。`index.html` を開く（または下のローカルサーバ）と最終確認できます。特に ドット絵シーン・吹き出し・2ゲージの見栄えは要・人の目チェック。

## 既知の調整余地（次に触るなら）

- 狂気のダメージ倍率（`kyokiDmgScale`）・ぬくもり閾値（`nukumoriStep`）は**暫定**。実機の手触りで詰める。
- 序盤（ウェーブ3〜4）の難度はやや高め。`healOnWaveStartPct` や `maxHp` で微調整可能。
- 手をのばすルートは「いっしょに かえろう」入手前がいちばん苦しい。`CARDS` の重みづけ（出現確率）導入も一案。
- 内部の関数名・データ id は v0.1 のまま（`cmdFight`/`WEAPONS`/`namida` など）。段階的なリネームは将来課題。
- 数値はすべて暫定。実機で遊んだ感触に合わせて `data.js` を詰めるのがおすすめです。
