# 品質向上バックログ（v0.x）

[GDD](./GDD.md) の品質目標「**一夜の体験としての密度**」に向けた初期タスク。担当別に整理。
監督が優先順位を決め、依存のないものは並列で進める。**まず実機プレイで現状把握**してから着手するのが推奨。

## designer（見た目・演出）
- [ ] 実機で UI 可読性を点検（フェーズ／選択肢／結果が一目で分かるか）
- [ ] コマンド実行時のフィードバック演出（祓い＝鋭い、救い＝やわらかい動き）
- [ ] 敵の「おだやか／敵対」状態を視覚的に明確化
- [ ] HP・こころ・救済率の計器デザイン（選択の蓄積が見える）
- [ ] 結末3種（しずか／にび／ひかり）のテーマ切替の見栄え向上

## developer（ロジック・基盤）
- [ ] 実機で回帰確認（既存機能が壊れていないか）
- [ ] 入力の手応え改善（誤連打防止・選択キャンセル・キーボード操作）
- [ ] カード出現の**重みづけ機構**を実装（game-designer の設計を受けて）
- [ ] 戦闘ログ表示の整理（情報量と読みやすさ）

## composer（音）
- [ ] 5 コマンドの SE を質感差つきで整備（祓い／救いの描き分け）
- [ ] 被弾・レベルアップ・結末の SE
- [ ] 夜の緊張 → 夜明けの解放へ向かう BGM

## writer（物語・文章）
- [ ] 敵＝「群れ」の設定とニュアンス付け
- [ ] ACT／武器／カードの名前・説明文の推敲（トーン統一）
- [ ] 戦闘ログ文の語り口を統一
- [ ] 3 結末の文章の情感を強化（断定せず委ねる）

## game-designer（数値・手触り）
- [ ] 序盤（W3–4）難度の調整（`healOnWaveStartPct` / `maxHp`）＋ JXA 測定
- [ ] カード出現重みの設計（救いルートの「引き運」依存を緩和）
- [ ] 祓い／救い両ルートの成立と結末分布を再測定

## 最初の一歩（推奨フロー）
1. 監督（メイン）＋オーナーで**実機プレイ**し、気になり点を洗い出す。
2. 出た課題を上の担当へ割り振り、優先度を決める。
3. 独立タスク（文章×音×数値）から並列で着手 → 統合 → 品質ゲート。
