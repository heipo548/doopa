## 1. ビジョン / ログライン

**ログライン:** ひと晩を生き延びろ。ただし、何を犠牲にするかは、あなたが選ぶ。

「さいごのともだち」は、**“強くなること” と “つながりを守ること” が両立しない一夜**を描くコマンド選択型サバイバーRPG。
プレイヤーは毎ターン「**祓う**（倒して即座に強くなる）」か「**救う**（時間をかけて誰も死なせない）」かの板挟みに置かれる。
その選択の総和が**夜明けの世界の姿**を決める——速く強くなるほど世界は空っぽに、慈悲を重ねるほど苦しいが温かい朝が来る。

**核となる狙い:** プレイヤー自身の倫理が、難易度と結末という**手触りとして返ってくる**こと。

## 2. ターゲット体験（感情曲線）

| 局面 | 感じてほしいこと |
| --- | --- |
| 序盤(W1–2) | 余裕のなさ。祓い（速い快感）への誘惑 |
| 中盤(W3–4) | 救いの難しさ。敵を「おだやか」にできた時の小さな安堵 |
| 終盤〜ボス | 自分の選択の蓄積が効く（救い特化＝生存力／祓い特化＝火力） |
| 夜明け | 「自分が何を選んだか」を結末で突きつけられる |

「正解」を与えない。**速い祓いも、苦しい救いも、どちらも正当**だと感じさせる。

## 3. コアループ

```
WaveStart（HP ひと息回復・こころ満タン）
  → PlayerTurn（コマンド選択） ⇄ EnemyTurn（おだやかな敵は攻撃しない）
  → Clear → LevelUp（3択カード）
  → 次の Wave … → Boss → Dawn（救済率で結末分岐）
```

全6ウェーブ＋ボスの「ひと晩」。1 ラン完結。

## 4. 主要メカニクス（死守する核を含む）

### 5 つのコマンド
- **祓う**：武器で攻撃して倒す。1ターンで強くなる（EXP多）。だが二度と会えない。
- **こころみる**：敵の「心の壁」を下げる。敵ごとに効く ACT が違う（手順パズル）。こころを消費。
- **すくう**：壁が 0 の相手を救う。救済+1・HP も少し回復。遅く弱いが、誰も死なない。
- **どうぐ**：なみだのおまもりで HP 回復（数量限定）。
- **まもる**：このターンの被ダメを大幅カット（全体攻撃のしのぎ）。

### こころ / 心の壁 / おだやか
- **こころ**：救済リソース。こころみる(1)・すくう(2)で消費。毎ウェーブ満タン。
- **心の壁(♥)**：救うのに必要な「こころみる」回数。効く ACT でだけ下がる。
- **おだやか**：壁が 0 になった敵は**もう攻撃しない**。＝「壁を下げる」こと自体が生存手段。

### ★ 非対称（これが企画の心臓・絶対に壊さない）
- **祓いルート**：速い・易しい・EXP 多・被弾減。だが喪失、救済率↓ → 結末 **しずかな朝**（最強だが空っぽ）。
- **救いルート**：遅い・難しい・こころ消費。だが救うと HP 回復＝持続。救済率↑ → 結末 **ひかりの朝**。
- 両ルートとも**成立**し、混成は中間（**にび色の朝**）。一方が支配的最適解になってはいけない。

### 成長
- ウェーブクリアで **3 択カード**（新武器・強化・最大HP/こころ・パッシブ）。
- 武器Lv最大＋対応パッシブで**進化**（なみだ砲→大こうずい／ぽよぽよ連打→ばくれつぽよ／ひかりの輪→救済の光）。
- **救済の光**（全体の壁を一気に下げる）が救いルートの要エンジン。

### 結末
- 救済率のしきい値で **しずか / にび / ひかり** に分岐（`ENDINGS[*].min`）。

## 4b. v0.2：ことばで、たたかう（口喧嘩バトル ＆ 語彙成長）

> 戦いを「武器で殴る」から **「ことばでぶつかる口喧嘩」** に寄せた。Undertale の FIGHT/ACT/MERCY を
> 「**きついことば / きいてみる / 手をのばす**」に翻訳。ちいかわ的な可愛さと“ことばの刃”の**ギャップ**が肝。

### 追加した核：**言えなかった言葉が、言えるようになる**
- 成長は「火力が上がる」だけでなく、**主人公が少しずつ ことばを覚え、言えることが増える**こと。
- 友を **手をのばして** むかえると、その子の“ことば”を教わる（`ENEMIES.gift` → `KIND_WORDS`）。
- 3択カードの「おぼえる：◯◯」でも語彙が増える。覚えた やさしいことばは **きいてみる でいつでも使える**。
- 結末では「**さいごに のこった ことば**」と「この夜 おぼえた ことば」を見せる。

### コマンド名（プレイヤーに見える表示／内部 id は据え置き）
| 旧（内部 cmd） | 新（表示） | 何をする |
| --- | --- | --- |
| 祓う（cmdFight） | **ぶつける** | きついことばを ぶつける。早く退けるが **狂気**が上がる |
| こころみる（cmdAct） | **きいてみる** | 相手の声をきく。心の壁を下げ、効くことばを探す |
| すくう（cmdSave） | **手をのばす** | 壁が消えた相手を むかえる。きずな＋・ことばを教わる |
| まもる（cmdDefend） | **こらえる** | ことばを飲み込んで耐える。被ダメ減 |
| どうぐ（cmdItem） | **もちもの** | おまもり等。HP回復 |

### 2つの雪だるまゲージ（ことばの積み方＝ビルド）
- **狂気**：きついことばで上がる。高いほど ことばダメージ↑（`kyokiDmgScale`）／画面が翳る。
- **ぬくもり**：やさしいことばで上がる。閾値ごとに **最大HP＋＆回復**（`nukumoriStep`）＝救いルートの雪だるま。
- 旧「救済の光」は **きついことばの反転進化「いっしょに かえろう」**＝狂気を積まず ぬくもり＋（救いビルドも“強く”なれる道）。
- **死守**：祓い（ぶつける）＝速い・狂気↑、救い（きいて→手をのばす）＝遅い・ぬくもり↑。この非対称は v0.1 から不変。

## 5. ナラティブ方針 → `writer`

- 敵＝**「群れ」**。倒す＝祓う、和解＝救う。敵は「悪」ではなく「分かり合えるかもしれない相手」。
- トーン：**説明しすぎない／余白を残す**。叙情的だが感傷に溺れない。
- 祓い＝喪失、救い＝つながり、という重みが**言葉から滲む**こと。
- 3 結末の文章は「夜明けに何が残ったか」を、断定せずプレイヤーに委ねる。

## 6. アートディレクション → `designer`

- **ドット絵 / ピクセルアートで再構築**（v0.2 方針）。リファレンスは「夜明け前の山道・小さな仲間の列・ろうそく/ランタンの灯り・静かな旅」。
  - 寒色の夜景に**暖色の灯り**を置く。色数はしぼる。背景は描き込みすぎず**シルエットと余白**で。
  - 背景シーン（`#scene`）：空のグラデ＋尾根3層＋ランタン＋小さな仲間の列を、**外部素材なしの CSS** で表現（`image-rendering: pixelated`）。
  - かわいいけれど少し不穏・切ない。**可愛さと“ことばの刃”の落差**を演出の主役に。
  - 可読性は**絶対に**犠牲にしない（UI は半透明パネルの上、シーンは後ろで“気配”に徹する）。
- **夜の不安 → 夜明けの解放**を色とテーマ切替で表現（結末ごとに世界の色が変わる）。
- 狂気が高いほど 彩度が落ち翳り、ぬくもりが高いほど 下から暖色がにじむ（積み方が見た目に出る）。
- 敵は簡易図形でよいが、**気配・状態（おだやか/敵対）**が伝わること。
- 言ったことばは**吹き出し**で見せ（カテゴリで色・質感が変わる）、救済率・HP・こころ・狂気・ぬくもりは「選択の蓄積」が見える計器に。

## 7. サウンド方針 → `composer`

- 合成音（Web Audio）のみ。外部音源を持たない。
- **祓い＝鋭い/強い、救い＝やわらかい/温かい**——音の質感で 2 つの道を描き分ける。
- 夜の緊張 → 夜明けの解放へ向かう BGM。自動再生制約に配慮（最初の操作後に開始）。

## 8. バランス哲学 → `game-designer`

- **データ駆動**：手触りは原則 `data.js` の数値だけで変える。
- 変更は**測定で裏付ける**（JXA 簡易 AI で到達率・救済率・結末分布）。
- 支配的最適解を作らない。**序盤がいちばん難しい**のは意図的だが、理不尽にしない。

## 9. 品質目標（PoC → クオリティ向上）

現状は **v0.1 PoC＝「コアが正しく動く」段階**。次の山は **「一夜の体験としての密度」**：

- 文章の情感（writer）／UI の可読性と演出（designer）／音の手応え（composer）／バランスの説得力（game-designer）。
- 技術的負債を増やさず、**各要素の調和**を上げる（バラバラに良くするのではなく、ひと晩の体験として統一）。

## 11. 非目標（やらないこと）

- ミニゲームの寄せ集めにしない。
- 複雑化のための複雑化をしない（メカニクスは「祓い vs 救い」を濃くするためだけに足す）。
- PoC 段階で v0.2 のメタ要素を混ぜない。
