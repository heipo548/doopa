## 1. 世界観

- **抽象**：可愛くポップな村（涼ラベンダーのちいかわ風）の中で、**主人公ルゥだけ**が、ひろう ことばの色で
  少しずつ翳っていく。可愛い世界に、一人だけ刃物のようなことばが混じる落差が肝。
- **具体**：
  - 入口はポップ：やわらかい村。プレイヤーは油断する。
  - 反転の引き金は会話分岐ではない：**ひろう言葉カード**と**口喧嘩での勝ち方**が、ルゥの人格を決める。
  - 凶暴な言葉で言い負かすほど、ルゥのセリフ・視界が翳る（`dark-1/2/3`、寒色化）。村は可愛いまま。
  - **必然性（監督要求）＝なぐれない村**：手を出すことが禁じられ、叩けない体になっている。だから争いは
    “ことば” でしか決着できない。相手はことばでぶつかり、こちらは言い負かすか寄り添うしかない。

## 2. ゲームシステム

- **抽象**：探索型 ADV ＋ ことばのカードバトル。ストーリーは会話選択ではなく、集めたカードと倒し方で傾く。
- **具体**：
  - **探索**：村の広場を歩き（クリック移動／キーボード）、NPC・かどっこ・セーブ点・立て札・出口に関わる。
  - **言葉カードの入手**：NPC に「どの言葉を知りたい?」と問われ、**3枚から1枚**を選ぶ（人格を傾ける最初の選択）。
  - **口喧嘩＝カードバトル（ターン制）**：相手は二つのゲージ。
    - **精神HP（見える / mindHp）**：harsh で削る。0で「言い負かした」＝**凶暴+**。
    - **思いやり（見えない / omoiyari）**：kind で満たす。満タンで「寄り添った」＝**優しさ+**。UI に一切出さない。
    - 敵ターンは固定圧でプレイヤーhpを削る（**非パーマデス**）。kind は一部 side（healSelf/lowerAtk）で耐える。
  - **分岐**：会話の選択肢ではなく、**勝ち方の蓄積**（lastWinKind / counters / tendency）で結果が決まる。
  - **操作計測 → 軽いメタ**：クリック数・スキップ・読み飛ばし・カーソル距離を裏で計測。結果画面で“軽い1行”だけ見透かす。
  - **セーブ/ロード**：localStorage（`kotoba_save_v1`）。metrics スナップショットも往復（“癖”が途切れない）。非パーマデス。

## 3. 面白さ

- **抽象**：**勝ち方がそのまま人格になる**。バトルを最適化パズルにせず、「どう勝つか」がルゥを染める一貫性。
- **具体**：
  - **毎ターンのジレンマ**：速く確実に削る（凶暴）か、手間をかけて思いやりを満たす（優しさ）か。
    思いやりは見えないので、急ぐ人ほど無意識に削る側＝凶暴へ流れる。
  - **入手の瞬間にも選択**：3択に harsh/kind を混ぜ、取る段階から人格を選ばせる。
  - **可愛さと刃物の落差**：世界は可愛いまま、ルゥの言葉だけが鋭くなる。
  - **見透かされる軽い戦慄**：意識した言葉だけでなく、無意識の操作の癖（読み飛ばし）を最後に そっと突きつける。

## 4. MVP スコープ

詳細は [`../../../docs/prototypes/04/MVP_SCOPE.md`](../../../docs/prototypes/04/MVP_SCOPE.md)。

- **IN**：タイトル → 村 → NPC会話 → 3択入手 → 口喧嘩1戦（両ルート勝利・非パーマデス）→ 結果（傾向＋軽いメタ）→ タイトル。最小セーブ。
- **OUT（後続）**：複数エリア/洞窟/分岐（裁判所・無人島）、多エンドのデータ化、フル神様メタ＋word→world演出、学校＝言葉ショップ購入、周回、ドット絵。

## 5. データ駆動（拡張の土台）

`js/data.js` が正典。以下を“データを足すだけ”で増やせる設計：
- **言葉カード**：`WORDS` に1件追加（kind/power[3]/levels[3]/side/flavor）。
- **NPC・敵**：`NPCS`/`ENEMIES` に追加（会話・3択・行動・gift）。
- **エリア/ノード**：`AREA.nodes` に追加（将来は複数 AREA へ）。
- **エンド/結果文**：`ENDINGS`（傾向比）と `META_LINES`（skip 連動）。
- **バランス**：`BALANCE`（winHarsh/winKind/useHarsh/useKind/darkenAt）。

## 6. バランス（MVP・検算済み）

| 項目 | 値 |
|---|---|
| プレイヤー hp | 28 |
| 敵 かどっこ | mindHpMax 34 / omoiyariNeed 36 / atk 3 / gift `tonari` |
| 開幕手札 | `urusai`(harsh) / `kiite`(kind) |
| 傾向重み | winHarsh=winKind=10（対称）・useHarsh3 > useKind2・pick±2 |
| 翳り | darkenAt [8,16,30]（harsh1戦で dark-2 まで／kind は 0 で翳らない） |

- **harsh ルート**：urusai(10)×4 → mindHp 34→0（4手）。被弾 atk3×3＝9、hp 28→19 生存。
- **kind ルート**：kiite(6)×6 → omoiyari 36 充填（6手）。被弾 atk3×5＝15、hp 28→13 生存（side 札でさらに余裕）。
- 非対称（harsh 4手 < kind 6手）成立、kind でも死なない（headless-check で検証）。

## 7. 状態遷移（FSM）

```
[起動] → TITLE ─(はじめる)→ FIELD(村) ─(NPC)→ DIALOGUE ─(終)→ CARDS(3択) ─(選ぶ)→ FIELD
                                   └─(かどっこ)→ BATTLE ─(勝つ)→ RESULT ─(タイトルへ)→ TITLE
        FIELD/BATTLE ─(☰/Esc)→ PAUSE        BATTLE ─(hp0)→ GAMEOVER ─(セーブから)→ FIELD（非パーマデス）
```
- 状態管理の中心は `state.js`（FSM）＋ `flags`（talked_npc / got_card / battle_done / area_mura_clear）。
- まず遷移の骨組みを一周つなぎ（core systems §0/§1）、セーブを早期に入れ（§2）、バトル1戦を完成（§3）、村1本を通す（§4）。
