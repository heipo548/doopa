# 04 — ルゥと せかいの あいだ

**ルゥと せかいの あいだ** ／ slug: `04-kotoba-no-android`
> ことばに ひとつ 足すと、せかいに なる。足りなかったのは、きみだ。

手を出せない村で 目を さました、ことばを ひとつも 持たない子「ルゥ」の、探索×ことばのカードバトル。
とげの ことばで **言い負かす** か、やわらかい ことばで **寄り添って収める** か——**どう勝ったかが、きみを 染める**。
2戦（かどっこ → くちなし）を経て、終盤『井戸の こえ』が きみの操作の癖まで見透かし、最後に
**ことば（WORD）に きみ（L＝ルゥ）を ひとつ 足すと せかい（WORLD）になる** ことに 気づく。

> このプロトは、添付3資料の「言葉を覚えるアンドロイド」コンセプトを、DOOPA AI Studio の
> マルチエージェント体制で **#03「WO_RD → WORLD」の“上位版”** として作り直したもの（同じコンセプト・別の物語/演出）。

> ビルド不要・依存パッケージなし。`index.html` をブラウザで開くだけで動きます（`file://` 可）。
> 外部素材なし（効果音は Web Audio 合成・絵は CSS / SVG のみ）。
>
> **核：見えない思いやりを満たすか、見える精神HPを削るか。急ぐ人ほど、無意識に削る側＝凶暴へ流れる。**
>
> 企画の詳細は [`docs/GDD.md`](./docs/GDD.md)。MVP の範囲・人間確認事項は [`../../docs/prototypes/04/`](../../docs/prototypes/04/)。

## このプロト固有のガードレール（壊さない3点）

1. **二つのゲージを攻めるバトル**
   - **精神HP（見える / mindHp）**：とげの ことば（harsh）で削る。0で「言い負かした」＝傾向 **凶暴+**。
   - **思いやり（見えない / omoiyari）**：やわらかい ことば（kind）で満たす。満タンで「寄り添った」＝傾向 **優しさ+**。
   - **思いやりゲージは UI に一切表示しない。** これが心臓。急ぐ人ほど、見えている精神HPを削る側＝凶暴へ無意識に流れる。
   - harsh＝速い（3〜4手で勝てる）が凶暴／kind＝遅い（小刻みに積む・6〜7手＝敵の言葉を多く浴びる）が優しい。**この非対称が核。**
2. **必然性＝なぐれない村**
   - この村は手を出すことが禁じられている＝叩けない。だから争いは “ことば” でしか決着できない。
   - 相手はことばでぶつかり、こちらは「言い負かす（harsh）」か「寄り添う（kind）」しかない。**だからバトルが起きる。**
3. **操作計測の神様メタ（井戸の こえ）＋ word→world / L の回収**
   - クリック数・テキストのスキップ／読み飛ばし速度・カーソル距離を裏で計測。
   - 終盤、神様的存在『井戸の こえ』が その計測を“鏡”として実値で提示し、見透かす。
   - 最後に **WORD＋L（＝ルゥ）＝WORLD** を回収（あだ名「ルゥ」は型番「L」の聞き間違い、という設定で統合）。

> 既定テーマは涼ラベンダーの村（#03「WO_RD → WORLD」の暖ピンクとは別パレット）。
> 凶暴が溜まると body に `dark-1/2/3` が付き、世界の色温度が寒色へ冷え、**主人公ルゥだけ**が翳る。

> ⚠️ 本プロトは **MVP（仮）**。仮タイトル・あだ名「ルゥ」・本名/伏線の扱い・配色・敵/NPC名・数値は
> すべて差し替え可能（データ駆動）。確定が要る論点は [`../../docs/prototypes/04/HUMAN_DECISION_QUEUE.md`](../../docs/prototypes/04/HUMAN_DECISION_QUEUE.md) に整理。

## 操作

操作は **マウス（クリック／タップ）中心**。キーボードでも遊べます。

| 操作 | 動作 |
|---|---|
| **「はじめる」** | プロローグ → 村へ。新規開始（計測スタート） |
| **「つづきから」** | セーブ（`kotoba_save_v1`）から再開。**癖（metrics）も復元**される |
| **広場をクリック／タップ** | ルゥが その場所へ あるく |
| **ノードをクリック（または近づく）** | こな/ばあ＝会話／がっこう＝ことばショップ（強い1枚を預かる）／かどっこ→くちなし＝口喧嘩／ことばの泉＝セーブ＆戦間の全回復／立て札＝テキスト |
| **キーボード** | 矢印/WASD で移動、Tab でノード選択、Enter/Space で会話送り・決定、Esc で中断メニュー |
| **会話をクリック** | 1文字ずつ表示中なら全文表示、表示後は次へ（**全文前に送る＝スキップ。裏で計測される**） |
| **バトルでことばを選ぶ** | きつい ことば（harsh）＝精神HPを削る／やさしい ことば（kind）＝思いやりを満たす（思いやりの数値は出ない） |
| **☰メニュー** | つづける／セーブ／音 ON-OFF／タイトルへ |

> hp が 0 になっても **パーマデスではない**：セーブから やりなおせる。
> 中ボス かどっこ を越えると くちなし（ボス）が現れ、撃破で **結果（傾向で分岐する3エンド）→ 神様メタ（井戸の こえ）→ word→world の回収 → タイトル**。

## 何を試したかったか（検証したい仮説）

- 「**勝ち方そのものが人格になる**」体験が、**1戦・5分**の最小スライスでも成立するか。
- **見えないゲージ**（思いやり）が、「急ぐ人ほど無意識に凶暴へ」という非対称を生むか。
- **暴力が禁じられた村**という必然性フレームが、口喧嘩バトルの動機として腑に落ちるか。
- 既存 #03 と**別ゲーム**として弁別できるか（配色・操作・呼称・必然性）。

## ファイル構成

```
04-kotoba-no-android/
├── index.html              # 画面コンテナ＋読込順配線
├── style.css               # 涼ラベンダーのパレット・全画面・HUD・fx・dark-N
├── README.md               # このファイル
├── docs/GDD.md             # 企画書（詳細）
├── js/
│   ├── data.js   # 正典（ことばカード/敵/NPC/村/結果/タイトル/数値）
│   ├── audio.js  # 最小SE（Web Audio 合成・任意）
│   ├── metrics.js# 操作計測（clicks/skip/dwell/cursor）
│   ├── state.js  # FSM・傾向2軸・フラグ・結末判定（DOM非依存）
│   ├── save.js   # localStorage 往復＋metricsスナップショット（kotoba_save_v1）
│   ├── battle.js # 口喧嘩バトル（見える精神HP/見えない思いやり・両ルート・非パーマデス）
│   ├── cards.js  # ことばカード3択
│   ├── field.js  # 村フィールド（ノード近接・出口ロック）
│   ├── ui.js     # 描画（思いやりは描かない）
│   └── main.js   # 配線・歩行ループ・遷移
└── tools/
    ├── headless-check.js   # DOM非依存コアの回帰（osascript -l JavaScript）
    └── ui-syntax-check.js  # DOM依存込みの読込・描画通し（同上）
```

検証（node 不要）:

```bash
cd prototypes/04-kotoba-no-android
osascript -l JavaScript tools/headless-check.js   # → "CORE OK"
osascript -l JavaScript tools/ui-syntax-check.js  # → "UI OK"
```
