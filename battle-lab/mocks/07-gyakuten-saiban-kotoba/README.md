# 07 — 言葉の矛盾を突く尋問バトル（逆転裁判 参考 / Contradiction Dialogue Trial）

Battle Lab のバトルモックのひとつ。**逆転裁判型の尋問アドベンチャー**が、この
「言葉を覚えていない無機質な主人公 L が、言葉で世界に干渉していく」世界観に
合うかを検証するためのモック。**完成版ではない。**

公開URL（GitHub Pages）:
**https://heipo548.github.io/doopa/battle-lab/mocks/07-gyakuten-saiban-kotoba/**

## 何を試したいか

> 相手の発言を読み、矛盾を見つけて、正しい証拠を突きつける体験が楽しいか。

敵を倒すのではなく、村長の言葉に隠れた**矛盾**や**本音**を見つけ、
「閉じた心」をほどくことを目的にしている。

## 遊び方

1. 村長の **発話ログ（証言）** を1行ずつ読む。
2. 気になる発言を **選ぶ**。
3. コマンド：
   - **たずねる** … その発言をもう一歩聞く。追加のやりとり・ヒントが出る。
     場合によって新しい発言が現れる（ノイズは減らない）。
   - **つきつける** … 記憶のかけら／言葉を、その発言に突きつける。
   - **考える** … いまの論点（軽いヒント）を表示する（ノイズは減らない）。
4. **正しい発言に正しい「記憶のかけら」**を突きつけると、閉じた心が1段階ほどける。
5. **間違った記憶**を突きつけると **ノイズ耐性 −1**。0で敗北。
6. **言葉**はどこに突きつけても「サブ反応」のみ（正解にはならない／ノイズも減らない）。
   試行錯誤のための安全な手札。
7. 3つの矛盾を解く（閉じた心 ■■■→□□□）と、**本音のページ**が開いて勝利。

### この案件の正解（ネタバレ）

| 証言 | 対象の発言 | 突きつける記憶 | 仕掛け |
|---|---|---|---|
| 第1証言 | 1. 昨日の朝まで知らなかった | 村長の手帳 | — |
| 第2証言 | 5. 鍵はたまたま持っていた | 学校の鍵 | **3番をたずねると5番が出現** |
| 第3証言 | 4. 村には関係ない | 子どもの落書き | — |

- **欠けた名札**は今回どの矛盾も解かない“赤いニシン”。雰囲気と将来拡張用に表示。
- 敗北＝死ではなく、L が「ほんとう」と「うそ」の区別を一時的に失う演出。

## ファイル構成

```
07-gyakuten-saiban-kotoba/
├─ index.html        画面の骨組み（読み込み順：data→engine→ui→main）
├─ style.css         見た目（パステル＋手帳/黒塗り/等幅Lの“少し不穏”）
├─ js/
│  ├─ data.js        証言3本・正解対応・反応テキスト・記憶/言葉・勝敗文（“内容”）
│  ├─ engine.js      進行ルール（DOM非依存の純粋ロジック・状態 state）
│  ├─ ui.js          描画（state を読んで DOM を組み直す）
│  └─ main.js        配線（クリック→engine→再描画）
└─ tools/
   ├─ headless-check.js  engine の回帰確認（JXA・ブラウザ不要）
   └─ dom-smoke.js       UI配線の煙テスト（擬似DOM上で ui/main を実行）
```

ロジック（engine）と内容（data）と見た目（ui）を分けてあるので、
証言者や証言を増やすときは基本 `data.js` を足すだけで済む。

## ローカルで動かす / 点検する

```bash
# そのまま開いても動く（依存ゼロ）
open index.html

# もしくはローカルサーバ経由（リポジトリ root から）
python3 -m http.server 8777 --bind 127.0.0.1
#  → http://127.0.0.1:8777/battle-lab/mocks/07-gyakuten-saiban-kotoba/

# 進行ルールの自動点検（46項目）
MOCKDIR="$PWD" osascript -l JavaScript tools/headless-check.js

# UI配線の自動点検（22項目）
MOCKDIR="$PWD" osascript -l JavaScript tools/dom-smoke.js
```

## 将来の拡張余地（構造だけ用意）

- 証言者を増やす（`TESTIMONIES` を相手ごとに分ける）
- 証言を複数ページにする／記憶のかけらを組み合わせる／発言同士の矛盾を突く
- 沈黙でしか開かない発言・本音のページを複数用意
- 言葉の3段階成長（威力ではなく“読める矛盾の種類”が増える方式）
  - 例：どうして？ → ほんとうに？ → それは、だれの言葉？
  - 例：それはちがう → いまの言葉は、ずれている → あなたは、本当はそう思っていない
