# 01 — Slay the Spire参考「言葉デッキバトル」

Battle Lab（バトルシステム試作場）の最初のモック。
**敵を倒すのではなく、言葉で「閉じた心」をほどいていく** カードバトルの検証用。

> これは完成版ではなく、バトルシステム検証用のモックです。
> 見た目の豪華さより「このバトル、面白くなりそうか？」を触って判断するためのものです。

## 遊び方

1. 毎ターン「こころ（=エナジー）」が3、手札（=今いえる言葉）を5枚引く。
2. カードをクリックすると こころ を使って効果が出る。
3. 相手（村長）の **次の感情**（行動予告）を見て、伝えるか・受けとめるかを選ぶ。
4. 「ターン終了」で相手の番へ。
5. 村長の **閉じた心** を0にすれば勝ち。自分の **ノイズ耐性** が0になると負け。

## 初期デッキ（語彙）

| 言葉 | コスト | 種別 | 効果 |
|---|---|---|---|
| こんにちは ×4 | 1 | つたえる | 閉じた心に 6 |
| だいじょうぶ ×4 | 1 | うけとめる | 受容 +5 |
| どうして？ ×1 | 1 | たずねる | 1枚引く＋相手にほころび1 |
| 沈黙する ×1 | 0 | とくべつ | 受容 +2／手札を1枚捨てる |

## 状態異常

- **受容** … そのターン、相手から受けるノイズを軽減（次の自分の番で0に戻る）
- **迷い** … 次に使う『つたえる』言葉のダメージ −2（1スタック消費）
- **ほころび** … 相手が次に受ける『つたえる』ダメージ ×1.5（1スタック消費）

## 敵：村長（閉じた心 42）

主人公を疑いながら試してくる。倒すべき悪者ではない。

- **疑う** … ノイズ8
- **問いかける** … ノイズ5 ＋ プレイヤーに迷い1
- **黙って見る** … その番は何もしない。次の攻撃が少し強くなる（+3）

## 拡張のしかた

- **カードを足す**: `data.js` の `CARD_LIBRARY` に定義を追加し、`STARTING_DECK` に枚数を足す。
- **敵を足す**: `data.js` の `ENEMIES` に追加し、`newBattle('id')` で呼ぶ。
- **新しい効果**: `engine.js` の `applyEffect()` に効果キーを1つ足す（データ駆動）。
- **言葉のレベルアップ**（将来）: 各カード定義の `levels:[]` を使う想定。
  例）ありがとう → 本当にありがとう → ありがとう、いてくれて
- **記憶のかけら（レリック）/ 学校（ショップ）/ エリア別ルール**: 今回は未実装。
  `data.js` にテーブルを足し、`engine.js` にフックを増やす形で拡張できる。

## バランス調整

`js/data.js` 先頭の `BALANCE` を触るのが一番速い（ノイズ耐性・こころ・ドロー枚数など）。

## 検証（Nodeなしで動く）

```sh
# このフォルダで実行
osascript -l JavaScript tools/headless-check.js
```

DOMに触れない `data.js`＋`engine.js` を結合し、勝ち筋と負け筋を最後まで自動で回して
エラーなく勝敗判定に届くかを確認する（ブラウザを開かずにロジックだけ点検）。
