# 15 — ことば選択型・理解バトル

**Battle 15：ことば選択型・理解バトル / The Words We Choose**

## 一言コンセプト

倒さない。毎ターン3つの「ことば」から選び、相手を**理解し**、警戒させすぎずに**心の距離を縮める**。

## このプロト固有のガードレール

- 攻撃力・防御力は無い。プレイヤースキル＝**相手の状態を読む力**。
- 理解度・警戒度は**数値を前面に出さない**（バーと「ことば」で見せる）。
- 失敗してもゲームオーバーにしない（相手が去る／表面的な会話で終わる、など物語の結果として扱う）。

## 操作

| 操作 | 動作 |
|---|---|
| 選択肢（3つ）をクリック | その「ことば／ふるまい」を選ぶ。理解度・警戒度・余裕・相手の状態が変化する |
| 相手の切り替え（上部） | ① 泣いている子（チュートリアル） / ② つよがりな門番 |
| ？あそびかた / ？ことば | チュートリアルと用語ヘルプ |
| ♪ BGM（右下） | BGMの ON/OFF（最初はOFF。押すと再生） |
| もう一度 / 次の相手へ | リスタート / 子に勝つと門番をすすめる |

## ルール（読み合いの軸）

- **理解度**：相手のことが分かった度合い。目標まで上げると本音が見えて、勝利。
- **警戒度**：相手の身構え。上限になると会話を**拒絶**して失敗。
- **余裕**：L 側の落ち着き。重い言葉ほど消費。0になると**焦り**、選べる言葉の質が落ちる。距離を取ると回復。
- **ターン制限**：制限内に理解度が届かないと、**表面だけの会話**で終わる。

### 選択肢の6カテゴリ
聞く / 受け止める / 伝える / 観察する / 距離を取る / 差し出す（回数制限）。

### 相手のタイプ（相性）
さびしがり / おこりっぽい / こわがり / つよがり。タイプで「効くカテゴリ／逆効果のカテゴリ」が変わる（複数タイプ可。**苦手が優先**）。

### 相手の状態
- **安心**：警戒が上がりにくく、聞く系が通りやすい（受け止める2連続で発生）。
- **沈黙**：選べる言葉が2つに減る。無理に聞くと警戒大。待つ・距離・差し出す・受け止めるで解ける。
- **強がり**：様子の文が信用できなくなる（「？」印）。観察や軽い言葉で崩れる。
- **動揺**：理解度の大チャンス。ただし踏み込みすぎると警戒も大きく上がる。

毎ターンの3択は「**安全だが小さい／高リスクで大きい／状況しだい**」の三角を基本に、相手のタイプ・状態・観察フラグ・ターン数から生成される（spec 8）。

## エンカウンター

- **① 泣いている子（チュートリアル）**：こわがり。いきなり聞きすぎると怖がる、待つ・受け止める・観察が効く、を自然に学ぶ。1ターン目は仕様どおりの三択（どうしたの？／大丈夫？／しゃがむ）。
- **② つよがりな門番**：つよがり＋さびしがり。最初は「強がり」状態。足元の跡を**観察**して「ずっと待っていた」ことに触れ、**差し出す**（失敗談・写真）で**動揺**を作り、本音を話せるところまで導く（spec 12）。

## 何を試したかったか

- 「強い行動を選べば勝てる」ではなく、**同じ言葉でも相手・状態・タイミングで届き方が変わる**読み合いが面白くなるか。
- 理解度・警戒度を数値で出さず「ことば」で見せても、判断材料として成立するか。
- ポケモン的な相性／状態 × Slay the Spire 的な毎ターン3択 × UNDERTALE 的な「観察して戦わず進める」を、ひとつの会話バトルに溶かせるか。

## 自動点検

進行ルール（engine）は DOM非依存なので、ブラウザを開かずに検証できる。

```bash
# macOS（JXA）
MOCKDIR="$PWD" osascript -l JavaScript tools/headless-check.js
MOCKDIR="$PWD" osascript -l JavaScript tools/dom-smoke.js

# Node（osascript の無い環境でも可。__dirname から親フォルダを解決）
node tools/headless-check.js
node tools/dom-smoke.js
```

`headless-check.js` は初期化・相性・効果量・状態遷移・余裕・3択生成・勝敗、両エンカウンターの勝ち筋、ランダム自動プレイの完走を点検する。`dom-smoke.js` は ui.js / main.js が参照する要素IDが index.html に存在するかを点検する。
