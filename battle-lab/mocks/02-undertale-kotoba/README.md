# 02 — UNDERTALE参考「言葉アクション対話バトル」

Battle Lab（バトルシステム試作場）の2つめのモック。
**敵を倒すのではなく、相手を観察し、言葉を選び、飛んでくる感情を避けながら「閉じた心」をほどく**——
UNDERTALE “参考” の対話×弾よけバトルの検証用。

> 英語表記：Word Action Dialogue Battle
> これは完成版ではなく、バトルシステム検証用のモックです。
> 見た目の豪華さより「このバトル、この世界観に合うか？」を触って判断するためのものです。
> ※ UNDERTALE そのものの見た目・音楽・固有表現はコピーせず、**構造だけ**を参考にしたオリジナルです。

## 遊び方

1. 4つの親コマンド（**つたえる / たずねる / おぼえる / しずまる**）から1つ選ぶ。
2. さらに小さな言葉の選択肢が出る。選ぶと **理解度・警戒度・ノイズ耐性** が変化し、村長がセリフを返す。
3. そのまま **感情回避フェーズ（約5.5秒）** に入る。
   矢印キー / WASD で「こころ」を動かし、**？！を避ける**。**◇（記憶のかけら）に触れると理解度+3**。
   スマホは canvas を指でドラッグ。
4. 回避が終わると次のターンへ。
5. **理解度が80以上**になったら、`しずまる → ほどく` で **勝利**。
6. **ノイズ耐性が0**になると **敗北**。

## ステータス

| 対象 | ステータス | 説明 |
|---|---|---|
| L（主人公） | ノイズ耐性 20/20 | 0で敗北。深呼吸で+3（最大20） |
| 村長 | 理解度 0〜100 | 80以上で「ほどく」が成功 |
| 村長 | 警戒度 0〜100 | 高いほど回避フェーズが少し激しくなる |
| 村長 | 閉じた心 | 理解度に応じて「まだ固い / 少しゆるんだ / ほどけそう」 |

## コマンド一覧

### つたえる（理解度を上げる）
| 言葉 | 効果 |
|---|---|
| こんにちは | 理解+8 / 警戒+3 |
| いやだ | 理解+12 / 警戒+10 |
| ありがとう | 理解+6 / 警戒-5 |
| 心配している（※`まねる`後に出現） | 理解+20 / 警戒-15 |

### たずねる（観察・質問。このモックの中心）
| 言葉 | 効果 |
|---|---|
| みる | 理解+5 / 警戒-3 |
| きく | 理解+8 / 警戒-8 |
| どうして？ | 理解+10 / 警戒+2 |
| まって | 理解+4 / 警戒-10 / 次の回避フェーズのノイズを少し遅く |

### おぼえる
| 言葉 | 効果 |
|---|---|
| まねる | 理解+5 /「心配している」を一時的に覚える（つたえるに追加） |

### しずまる（回復・防御・勝利）
| 言葉 | 効果 |
|---|---|
| 沈黙する | 警戒-5 / 次の回避フェーズの被ダメを軽減 |
| 深呼吸する | ノイズ耐性+3（最大20） |
| ほどく | 理解80以上で勝利。未満だと「まだ、ほどけない」 |

## 感情回避フェーズ（弾よけ）

村長の「次の感情」に応じて、飛んでくるノイズの傾向が変わる。

| ノイズ | 見た目 | 動き | 効果 |
|---|---|---|---|
| 疑いノイズ | ？ | ゆっくり飛ぶ | 当たると 1 ダメージ |
| 怒りノイズ | ！ | 直線的に速く飛ぶ | 当たると 2 ダメージ |
| 記憶のかけら | ◇（こ・と・ば…） | ゆっくり落ちる | 触れると 理解度 +3 |

被弾後は短い無敵時間がある（耐性20なので連続被弾で即終わらない配慮）。

### 村長の感情パターン
| 感情 | セリフ | 弾幕 |
|---|---|---|
| 疑う | 村長は、あなたを疑っている。 | ？中心 |
| 問い詰める | 村長「おまえは、何をしに来た」 | ！多め |
| 黙って見る | 村長は、何かを言いかけてやめた。 | 少なめ＋かけら少し |
| 心配する（理解60以上で出現） | 村長は、あなたを怖がっているのではない。心配している。 | 少なめ＋かけら多め |

## ファイル構成

```
02-undertale-kotoba/
├─ index.html       # 画面の骨組み（要素ID・コマンド枠・弾よけcanvas・勝敗オーバーレイ）
├─ style.css        # 見た目（白基調＋パステル / L=無機質 / 村長=かわいいが警戒）
├─ js/
│  ├─ data.js       # ★コマンド木・効果・感情・ノイズ・バランス定数（増やすならここ）
│  ├─ engine.js     # コマンドフェーズの進行（理解/警戒/耐性の変化・勝敗。DOM非依存）
│  ├─ bullet.js     # 感情回避フェーズの弾幕模型（生成/移動/当たり判定。DOM非依存）
│  ├─ ui.js         # 描画（DOMとcanvasはここだけ）
│  └─ main.js       # 配線（クリック→engine→再描画、rAFループ、キー/タッチ入力）
└─ tools/
   └─ headless-check.js  # JXAでの自動検証（効果値・勝敗・当たり判定・オートラン）
```

## 拡張のしかた（将来の構造だけ用意済み）

- **言葉を足す**：`data.js` の `COMMAND_TREE` に option を追加（effect はデータで書ける）。
- **新しい効果**：`engine.js` の `applyCommand()` に effect キーを1つ足す（データ駆動）。
- **相手ごとの専用セリフ / 複数の勝利ルート**：`COMMAND_TREE` と `WIN_TEXT` を相手別テーブルに広げる。
- **感情ごとの弾幕パターン**：`data.js` の `EMOTIONS[*].spawn` を増減、`NOISE_TYPES` に弾を追加。
- **言葉のレベルアップ（3段階）**：例）まって → 少し待って → あなたの言葉を待ってる。
  option に `levels:[]` を持たせて段階表示する想定。
- **学校＝言葉ショップ / エリア別バトル（村・洞窟・裁判所・無人島）**：今回は未実装。

## 検証（Node なしで動く）

```sh
# このフォルダで実行
osascript -l JavaScript tools/headless-check.js
```

DOM に触れない `data.js`＋`engine.js`＋`bullet.js` を結合し、
効果値・クランプ・勝敗・当たり判定・勝ち筋/負け筋の到達性を自動で点検する（ブラウザを開かずにロジックだけ確認）。
