# 04 — Dicey Dungeons参考「言葉サイコロ装填バトル」

Battle Lab（バトルシステム試作場）の4つめのモック。
**毎ターン出る「ことばの欠片」を「言葉装置」に装填して効果を出す**、Dicey Dungeons 型バトルの検証用。

> これは完成版ではなく、バトルシステム検証用のモックです。
> Dicey Dungeons そのものの見た目・音・固有表現はコピーせず、**構造だけ**を参考にしたオリジナルです。
> いちばん試したい手触り：「Lはまだ言葉を完全には知らない。だから、その場で出た欠片で なんとか言葉にする」。

## 遊び方

1. 毎ターン「ことばの欠片」が3つ出る（各 1〜6 の「音の強さ」）。
2. **欠片をクリックして選び**、入れたい**言葉装置をクリック**すると装填できる（ドラッグ不要）。
3. 装置ごとの**条件**（任意／奇数／偶数／4以下／4以上／同じ音…）を満たせば効果が出る。
4. 村長の **次の感情**（予告）を見て、受容する（守る）か・伝える（閉じた心を削る）かを選ぶ。
5. 使わなかった欠片はターン終了で消える。「ターン終了」で相手の番へ。
6. 村長の **閉じた心** を0にすれば勝ち。自分の **ノイズ耐性** が0になると負け（＝言葉がばらばらになる）。

## 言葉装置（8個：必須5＋任意3）

| 装置 | 条件 | 効果 |
|---|---|---|
| こんにちは | 任意の欠片1つ | 閉じた心 −欠片の音 |
| だいじょうぶ | 任意の欠片1つ | 受容 ＋欠片の音 |
| どうして？ | 奇数（1・3・5）1つ | 閉じた心 −2／次ターン欠片＋1 |
| まって | 偶数（2・4・6）1つ | 次に受けるノイズ −3 |
| 沈黙する | 欠片を1つ捨てる | 受容 ＋2（捨てた音が1なら ＋5） |
| ありがとう（任意） | 4以下の欠片1つ | 受容 ＋欠片の音／次のノイズ −1 |
| いやだ（任意） | 4以上の欠片1つ | 閉じた心 −8／自分にノイズ ＋2 |
| ほんとう？（任意） | 同じ音の欠片2つ | 閉じた心 −10／村長の感情を引き直す |

- 条件に合わない欠片は装置に入らず、「この欠片では言葉にならない」と表示される。
- **白い欠片**：どの条件にも使えるワイルドカード（音3）。使うと追加で受容＋1。
- **かすれ**：村長の「問いかける」を受けると、次ターンの欠片が1つ「音1」に固定される。

## 敵：村長（閉じた心 30）

悪者ではない。あなたを疑い、怖がり、でもどこか心配している。毎ターン「次の感情」を予告する。

| 感情 | 効果 |
|---|---|
| 疑う | 6ノイズ |
| 問いかける | 4ノイズ ＋ 次ターンの欠片を1つ「かすれ」に |
| 黙って見る | 何もしない。次の感情が少し強くなる（溜め） |
| 心配する | 3ノイズ ＋ 次ターン白い欠片を1つくれる（閉じた心が低い／低確率で出る） |

## 受容（ブロック）

そのターン、村長から受けるノイズを軽減する。敵ターンが終わると0に戻る。
（例：ノイズ6・受容4 → 実際に受けるノイズ2）

## ファイル構成

```
04-dicey-dungeons-kotoba/
├─ index.html       # 画面の骨組み（要素IDとHUDラベル）
├─ style.css        # 見た目（村=ポップ / 主人公L=無機質）
├─ js/
│  ├─ data.js       # ★言葉装置・敵の感情・欠片・バランス定数（増やすならここ）
│  ├─ engine.js     # バトル進行（DOM非依存＝JXAで検証できる純粋ロジック）
│  ├─ ui.js         # 描画（DOMはここだけ）
│  └─ main.js       # 配線（欠片を選ぶ→装置に入れる→再描画／選択状態の管理）
└─ tools/
   ├─ headless-check.js  # JXA：data+engine の自動検証（各装置の条件・効果・勝敗）
   └─ dom-smoke.js       # JXA：ui+main まで通す煙テスト（描画・操作・勝利描画）
```

## 拡張のしかた

- **言葉装置を足す**: `data.js` の `DEVICE_LIST` に1件追加するだけ（`accept` 条件と `effect` を指定）。
- **新しい条件**: `engine.js` の `kakeraFits()` に `accept` の分岐を1つ足す。
- **新しい効果**: `engine.js` の `fireDevice()` に効果キーを1つ足す（データ駆動）。
- **相手を足す**: `data.js` の `ENEMIES` に追加し、`newBattle('id')` で呼ぶ。
- **欠片の種類**: 白い欠片・かすれ欠片の仕組みを土台に、文字化け欠片などを足せる。
- **言葉のレベルアップ**（将来）: 各装置定義の `levels:[]` を使う想定（例：ありがとう → 本当にありがとう → ありがとう、いてくれて）。

## バランス調整

`js/data.js` 先頭の `BALANCE` を触るのが一番速い（ノイズ耐性・閉じた心・毎ターンの欠片数・溜め量など）。

## 検証（Nodeなしで動く）

```sh
# このフォルダで実行
osascript -l JavaScript tools/headless-check.js   # ロジック（data+engine）
osascript -l JavaScript tools/dom-smoke.js        # UIまで（ui+main）
```
