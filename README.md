# DOOPA — ドーパミン直接摂取マシーン

> "ゲーム" ではなく、ゲームの一番気持ちいい瞬間だけを連続摂取する装置。

## コンセプト

### ターゲット
**ドパガキ** = TikTok / YouTube Shorts によって常に新しい刺激を求める状態にされた若者。
- 同じコンテンツに飽きるのが早い
- 本は落ち着いて読めない
- 映画は長くて最後まで見れない
- 「ゲームを覚えて上手くなる」のは面倒くさい

### 既存ゲームとの違い

| | 普通のゲーム | ミニゲーム集 (Roblox等) | **DOOPA** |
|---|---|---|---|
| 単位 | ステージ・レベル | ゲーム | **刺激（3〜10秒）** |
| ルール理解 | 必要 | 必要 | **不要** |
| 失敗 | ある | ある | **ない（全部報酬）** |
| クリア | ある | ある | **ない（飽きたら次へ）** |
| アルゴリズム | なし | なし | **TikTok的に最適化** |

### キーアイデア
**「ゲームの本質はドーパミン摂取である」** という仮説をそのまま形にする。
ゲームから「ストーリー」「ルール理解」「上達」「失敗」を全部抜く。
残るのは **破壊・収集・連打・レア発見・成長** といった "気持ちいい瞬間" のエッセンスだけ。
それを TikTok 的な縦スワイプで連続摂取する。

「ゲームをやってる」というより「ドーパミンを直接静脈注射してる」感覚を目指す。

### 風刺としての側面
ターゲットである "ドパガキ" 自身を観察対象にしたメタゲーム。
プレイヤーは遊んでるうちに「自分がこんな単純な刺激で気持ちよくなれてしまう」ことに気づく。
（気づかずに無限に遊んでも、それはそれで成立）

## 操作

| 操作 | アクション |
|---|---|
| `↓` キー / マウス下ドラッグ | 次の刺激へ |
| `↑` キー / マウス上ドラッグ | 前の刺激に戻る |
| `L` キー / 💖 ボタン | いいね（次に似た刺激が出やすくなる） |
| `D` キー / 👎 ボタン | やだ（次に出にくくなる） |
| `Space` キー | 一時停止 / 再開 |

各刺激内の操作（タップ等）はマウスクリックです。

## 初期搭載の刺激（5種）

| 刺激 | カテゴリ | 操作 | 体感時間 |
|---|---|---|---|
| 🎈 風船パンッ | 破壊 | クリック連打 | 6秒 |
| 🔢 数字爆増 | 成長 | クリック | 5秒 |

## 今後の発想拡張アイデア（朝起きたら相談したいやつ）

実装はしてないが、コンセプトを広げるなら次の方向性がある：

1. **「飽き」を検知して進化する刺激** — 同じ刺激でも、プレイヤーが飽きると次回は派手さが増す / ルールが変異する
2. **プレイヤーの「ドーパミン履歴」を可視化** — このユーザーはこういう刺激が好き、を見せる（怖さも含めて）
3. **2人で同時スワイプ** — 隣の人と同じ刺激を引き当てたら大演出（ガチャの当たり共有）
4. **刺激のミックス** — 風船+コイン雨など、2つの刺激を同時にぶつける
5. **物理キーボードへの依存を減らす** — マウスホイール・トラックパッドのスワイプにも対応
6. **「もう寝ろ」モード** — 一定時間プレイしたら画面が暗くなり、強制終了。中毒批判を内包
7. **SNSシェア導線** — プレイヤーが摂取した刺激ログを画像化して共有

## 技術メモ

- 純粋な HTML/CSS/JavaScript（ES Module 不使用、`<script>` タグ直読み）
- ローカルファイル（`file://`）で動く前提、サーバー不要
- 音は Web Audio API でリアルタイム生成（外部音源ファイルなし）
- ビジュアルは CSS アニメーション + Canvas（Canvas は使う刺激のみ）
- 依存ライブラリゼロ
