# さいごのともだち 開発スタジオ（エージェント運用ガイド）

このフォルダは「さいごのともだち」（`prototypes/02-saigo-no-tomodachi`）を**複数のエージェントで分業して品質を上げる**ための定義集です。
> 現状は本作専用。将来ほかのプロトにも使う場合は、汎用化するか別途用意するかを監督が判断します。

## メンバー

| 役割 | スラッグ | 主担当 | 触るファイル |
| --- | --- | --- | --- |
| 監督 | `director` | 企画の番人・割り振り・統合・品質判定 | （実装はしない） |
| デザイン | `designer` | UI/UX・見た目・演出 | `style.css` / `index.html` / `ui.js`(描画) |
| 開発 | `developer` | ロジック・FSM・入力・検証 | `js/state,battle,cards,main` / `ui.js`(入力) |
| 音楽 | `composer` | SE/BGM・音の演出 | `js/audio.js` |
| ストーリー | `writer` | 世界観・セリフ・結末文 | `data.js`(文字列) / 文言 |
| ゲームデザイン | `game-designer` | 数値・難易度・カード | `js/data.js` |
| テストプレイ | `dopagaki` | 初見目線の手応え検証・つまずき洗い出し | （実装はしない・症状を担当へ振る） |

**唯一の基準＝企画書:** [`../../prototypes/02-saigo-no-tomodachi/docs/GDD.md`](../../prototypes/02-saigo-no-tomodachi/docs/GDD.md)

## オーケストレーションの回し方

1. **監督はメインの会話（あなたと話している Claude）が兼任**します。Claude Code のサブエージェントは入れ子で他を呼べないため、`director.md` は監督の**判断マニュアル**として使い、実際の委譲はメインが `Agent` ツールで行います。
2. 要望を GDD に照らして**要件分解** → 役割表で**担当を決定** → **依存のない作業は並列**で委譲。
3. 上がった成果を**統合**し、**品質ゲート**（動作／企画整合／一貫性／ルール遵守）を通す。通らなければ差し戻し。
4. 完了したら README / GDD / バックログを**最新化**。

### 並列の目安
- 同時に動かしやすい：`writer`（文章）× `composer`（音）× `game-designer`（数値）。
- 競合しやすい：`designer` と `developer`（`ui.js`/`index.html` を共有）→ 監督が順番か担当範囲を整理。
- `dopagaki`（テストプレイ）は**最初**（現状把握）と**最後**（統合後の手応え確認）に回すと効く。実装はしないので他と競合しない。

### 3本柱（GDD の芯・全員が忘れない）
**Undertale**（祓う/救うの選択と「救いの意味」）×**ダダサバ/ヴァンサバ**（強くなる爽快感・ビルドの雪だるま）×**ちいかわ**（かわいくて、どこか切ない手触り）。

## 使い方（重要：認識のための起動）

これらは**プロジェクト直下の `.claude/agents/` でのみ自動認識**されます。今 Claude Code はホームで起動しているため、**doopa フォルダで起動し直す**必要があります。

```sh
cd ~/Projects/ai-workspace/99_sandbox/doopa
claude
```

起動後は、メイン会話が `Agent` ツールで `subagent_type` に各スラッグ（例 `developer`）を指定して委譲します。description に沿って自動委譲されることもあります。
> セッション途中で追加した定義は、起動し直すまで認識されないことがあります。
